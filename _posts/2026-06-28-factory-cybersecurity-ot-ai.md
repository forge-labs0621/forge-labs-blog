製造業の工場では、生産ラインを制御するOT（Operational Technology: 制御系）システムへのサイバー攻撃が増えています。ランサムウェアによる生産停止・情報漏洩のリスクに対し、AIを使ってセキュリティ対策書類の整備を効率化できます。

---

## 工場のOTセキュリティリスクアセスメントを整理するプロンプト

工場の制御システム・ネットワークのセキュリティリスクを整理します。

```
以下の情報をもとに、工場のOT（制御系）セキュリティリスク
アセスメントを整理してください。

【自社の工場システム構成（例）】
OT側（制御系）:
  ・PLC（プログラマブルロジックコントローラ）: ○台
  ・SCADA/HMI（監視制御システム）: 生産ライン管理用1台
  ・設備のEthernet接続: 一部の工作機械がLANに接続されている
  ・産業用IoTセンサー: 温度・振動センサーをルーターで収集

IT側（情報系）:
  ・社内LAN（Windows PC）
  ・クラウド経由の生産管理システム（MES）
  ・インターネット接続あり

OT/ITの境界:
  ・OTとITが同一ネットワークで接続されている（分離できていない）

【主なセキュリティリスク】
・OT/ITが分離されていないため、IT側のマルウェアがOT側に伝播するリスク
・工場に持ち込まれるUSBメモリからのウイルス感染
・メンテナンス用のリモートアクセス（VPN）の認証が弱い
・設備メーカーのサポート用リモート接続が常時有効になっている

出力形式:
・「OTセキュリティリスクアセスメント表（資産×脅威×脆弱性×リスク）」
・「優先的に対処すべきリスクのランキングと根拠」
・「中小製造業がまず取るべきOTセキュリティ対策5選」（コスト低め・効果大）
・「IPA『中小企業の情報セキュリティ対策ガイドライン』との対応確認表」
```

## ランサムウェア感染時の対応計画を作るプロンプト

工場PCがランサムウェアに感染した場合の初動対応を整理します。

```
以下の情報をもとに、製造業工場でのランサムウェア感染時の
対応計画（インシデントレスポンスプラン）を作成してください。

【想定シナリオ】
・朝8時に出社したら、事務所PCの複数台でファイルが暗号化されており
  「身代金を支払わないとデータを公開する」というメッセージが表示された
・生産ラインのSCADA端末は現時点では正常に動いている
・社内ネットワーク全体に感染が広がっているか不明

【社内の対応リソース】
・IT担当者: 1名（兼任）
・情報セキュリティの専門家: 社内にいない
・保険: サイバー保険未加入
・バックアップ: 毎週末に外付けHDDにバックアップ（最終は3日前）

出力形式:
・「ランサムウェア感染発覚後の初動対応フロー（最初の1時間）」
  （ネットワーク切断・感染範囲確認・報告連絡先の優先順位）
・「外部への連絡先リスト（警察（都道府県警察サイバー犯罪相談窓口）・
  IPAセキュリティセンター・取引先・顧客への連絡タイミング）」
・「身代金を支払うべきか・支払わない場合の選択肢」の整理
  （支払わない場合の対応: 復旧手段の検討）
・「再発防止策の優先順位（バックアップの強化・ネットワーク分離・
  EDRツール導入等）」
・「ランサムウェア対応後の顧客・取引先への説明文の骨子」
```

## 情報セキュリティポリシーの整備プロンプト

製造業としての情報セキュリティ基本方針と規程を整理します。

```
以下の情報をもとに、製造業の情報セキュリティポリシー
（基本方針・規程・ルール）の骨子を整理してください。

【自社の状況】
・従業員数: ○名 / 工場と本社が同一敷地
・情報セキュリティ規程: なし（暗黙のルールはあるが文書化なし）
・最近の懸念事項:
  ① 退職した従業員が社内資料を持ち出した
  ② 取引先からセキュリティ診断への回答を求められた
  ③ 顧客から「情報セキュリティの取り組みを教えてほしい」と言われた

・認証取得の予定: Pマーク・ISMS取得は現時点で検討外
  （まず「最低限の書類整備」を優先）

出力形式:
・「情報セキュリティ基本方針（1ページ）のドラフト」
  （経営層の署名が入る形式）
・「社員向けの情報セキュリティルール（A4 1枚・箇条書き10項目程度）」
  （PC管理・パスワード・USBメモリ・メール・社外秘の持ち出し等）
・「退職者のアカウント・アクセス権限抹消チェックリスト」
・「取引先・顧客からのセキュリティ質問票への回答テンプレート」
  （よく聞かれる項目への標準回答文）
```
