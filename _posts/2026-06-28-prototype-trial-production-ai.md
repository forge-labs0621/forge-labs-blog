新製品の試作から量産移行には、多くの書類作成と確認作業が伴います。「試作管理台帳」「量産移行チェックリスト」「初物確認書（ファーストアーティクル）」などの文書化をAIで効率化できます。

---

## 試作報告書・試作結果サマリーを作るプロンプト

試作が完了した後に提出する試作報告書の文章を整理します。

```
以下の情報をもとに、試作報告書を作成してください。

【試作の基本情報】
試作品番・品名: （例：プレスブラケット 試作品番 PROTO-A1234）
試作目的: （例：新規取引先○○社への量産受注にあたる設計妥当性確認）
試作回数: （例：第1回試作（Phase 1））
試作数量: （例：10個）
試作実施日: （例：2026年7月10〜12日）
担当: （例：製造部 ○○、品質保証部 ○○）

【試作の結果】
寸法確認結果: （例：10個全数を測定。図面Rev.Aのすべての寸法項目が公差内。
  特に重要寸法（フランジ外径φ50±0.05mm）は10個全数で合格）
外観確認結果: （例：バリ・傷なし。表面粗さは全数良品）
機能確認結果: （例：客先治具への組み付け確認OK。干渉なし）
発見した課題: （例：バリ取り後の洗浄工程で油残りが1個発生。作業手順の
  見直しが必要）

【次フェーズへの申請】
次のアクション: （例：試作1回で設計妥当性を確認。2026年8月よりQDR
  （量産前品質レビュー）に進む）

出力形式:
・試作報告書フォーマット（目的・方法・結果・課題・次アクション）
・試作数量・合格数量・不合格数量（あれば）の表
・承認欄・確認日（プレースホルダー）
```

## 量産移行チェックリストを作るプロンプト

試作完了から量産開始前に確認すべき事項をチェックリスト化します。

```
以下の製品情報をもとに、量産移行チェックリストを作成してください。

【製品情報】
品番・品名: （例：精密ブラケット A-1234、客先: ○○自動車工業）
量産開始予定: （例：2026年9月1日）

【量産移行前に確認が必要な領域】
- 設計・図面面: 量産図面（Rev.C）の発行・配布
- 製造面: 作業標準書の作成・設備段取り条件の確定
- 品質面: 量産検査基準書の発行・ゲージ類の準備
- 材料・調達面: 量産用材料の発注・在庫確認
- 外注面: 外注先への量産移行通知・試作結果の共有
- 顧客承認面: 初物（ファーストアーティクル）の提出・承認取得
- ISO 9001 記録: 4M変更管理記録の発行（新規製品として登録）

出力形式:
・「確認項目 / 担当部門 / 完了期限 / 完了確認（記入欄） / 備考」の表形式
・「必須」（量産開始のブロッカー）と「推奨」（望ましい）を分類して表示
・最終承認欄（量産移行決定者のサイン欄）
```

## 初物確認書（ファーストアーティクル）を作るプロンプト

量産初品（初物）を客先・社内に提出する際の確認書を作ります。

```
以下の情報をもとに、初物確認書（ファーストアーティクル確認書）を
作成してください。

【製品情報】
品番・品名: （例：精密ブラケット A-1234）
ロット番号: （例：LOT-20260901-001（量産初品））
数量: （例：50個（初物として10個を確認用に提出）））

【確認項目と結果】
寸法確認: （例：図面Rev.C全12箇所の寸法を測定。全数合格。
  測定成績書を別紙添付）
外観確認: （例：10個全数目視検査。バリ・傷なし。合格）
材質確認: （例：ミルシートで材質確認済み（SECC）。合格）
機能確認: （例：客先提供の治具で組み付け確認OK）

【提出内容】
（例：
・初物確認書（本書）
・全数寸法測定成績書（別紙1）
・材質証明書（ミルシート）（別紙2）
）

出力形式:
・初物確認書フォーマット
・「承認」「条件付き承認」「否認」の判定欄（プレースホルダー）
・製造者・品質保証者の確認サイン欄
・「この初物確認が承認された後、量産を開始します」の注記
```

## 試作管理台帳を更新するプロンプト

試作の進捗を一覧で管理する台帳の記載を整理します。

```
以下のメモをもとに、試作管理台帳の記載内容を整理してください。

【試作管理台帳に追加する情報】
品番: PROTO-A1234
試作段階: 第1回試作完了・第2回計画中
担当者: 製造部 ○○、品質保証部 ○○
試作1回目の状況:
  - 実施日: 2026年7月10〜12日
  - 数量: 10個
  - 結果: 寸法・外観は全数合格。油残り問題を1件検出・改善対応中
  - 次フェーズ: 2回目試作（改善確認）2026年8月予定

出力形式:
・台帳の1行分として管理できる情報整理
・「試作回数 / 実施日 / 試作数 / 合否 / 課題 / 次アクション / 担当」のフォームで
```
