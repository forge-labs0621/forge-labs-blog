自動車・電子機器など品質要求の厳しい業界では、取引先からのクレームや品質不良が発生した際に「8D報告書（8 Disciplines）」または「なぜなぜ分析」の提出を求められることがあります。内容は現場の事実確認が必要ですが、報告書の「文章の構成・整理」はAIが大きく手伝えます。

> **重要**: 8D報告書・なぜなぜ分析の内容（事実・原因・対策）は現場での調査結果に基づくものです。AIは「文章の構成・整理・コメントの清書」を補助するものであり、根本原因や恒久対策の判断は必ず技術者・品質担当者が行ってください。

---

## 8D報告書とは

8D報告書（8 Disciplines Problem Solving）は、製造業でクレーム・品質問題への対応プロセスを8つのステップで記録する書類です。

| ステップ | 内容 |
|---|---|
| D1 | チーム編成（問題に対応するメンバー） |
| D2 | 問題の定義（何が・どこで・いつ・どのくらい） |
| D3 | 暫定処置（応急対応） |
| D4 | 根本原因の特定 |
| D5 | 恒久対策の立案 |
| D6 | 恒久対策の実施・検証 |
| D7 | 再発防止・標準化 |
| D8 | チームへの感謝・終結 |

## 8D報告書の骨子を整理するプロンプト

現場で確認した事実・原因・対策をもとに、8D報告書の各ステップを整理します。

```
以下の情報をもとに、8D報告書の各ステップの文章を整理してください。

【D2: 問題の定義】
- 不良の内容: （例：プレス加工品のフランジ部にバリが発生。高さ0.8mm以上のバリが
  検査合格品に混入していた）
- 発生数量: （例：取引先○○社の受入検査で12個中3個が不良判定）
- 発見場所: （例：取引先○○社の受入検査ライン）
- 発見日: （例：2026年7月15日）
- 影響範囲: （例：ロット番号LOT-20260710-001の200個。出荷済み・一部は組立ライン
  に投入された可能性あり）

【D3: 暫定処置（実施済み）】
- （例：(1) 同ロットの在庫品を全数回収し再検査実施（200個中15個を不合格処分）
  (2) 取引先ラインに投入済みの品について、全数確認を依頼
  (3) 当面の出荷を停止）

【D4: 根本原因（調査結果）】
- 直接原因: （例：バリ取り工程での手仕上げ基準が不明確で、担当者によって
  確認基準にばらつきがあった）
- 流出原因: （例：出荷前の全数検査で0.5mm以上を基準としていたが、
  取引先図面の要求は「バリなし（0.3mm以下）」だった。図面確認の手順に抜けがあった）

【D5/D6: 恒久対策（決定・実施済み）】
- （例：(1) バリ取り検査基準を図面要求値（0.3mm以下）に改訂し
  作業標準書を更新（2026-07-20実施）
  (2) 出荷前検査に「図面要求値との照合」ステップを追加（同日）
  (3) 全作業者への教育（2026-07-22実施）

【D7: 再発防止・標準化】
- （例：類似品番（B-2345, C-1111）の検査基準も同様に見直し。
  年1回の検査基準レビューをQC会議の議題として追加）

出力形式:
・D1〜D8の各ステップを番号付きで
・D1はチーム編成として「品質保証部・製造2課・生産技術」と入れてください
・D8は「関係者への感謝と終結宣言」の定型文で
・各ステップは3〜5行（報告書として読みやすい量）
```

## なぜなぜ分析の整理を補助するプロンプト

現場で特定した「なぜ」を整理して、根本原因まで論理的につながる分析文を作ります。

```
以下のなぜなぜ分析の内容を整理して、論理的なつながりを確認してください。

【問題事象】
（例：取引先への出荷品にバリ（0.5mm超）が混入していた）

【なぜ1〜5の回答（現場が調査した内容）】
なぜ1: バリが製品に残っていたのはなぜか？
→ バリ取り作業後の確認が不十分だったため

なぜ2: バリ取り後の確認が不十分だったのはなぜか？
→ 検査基準書に「バリ高さ0.5mm以下」とあったが、図面の要求は「0.3mm以下」で
  担当者は旧基準で判断していたため

なぜ3: 担当者が旧基準で判断していたのはなぜか？
→ 図面改訂時に検査基準書の更新が行われなかったため

なぜ4: 図面改訂時に検査基準書が更新されなかったのはなぜか？
→ 図面改訂の手順書に「関連する検査基準書の確認・更新」のステップがなかったため

なぜ5: 図面改訂の手順書にそのステップがなかったのはなぜか？
→ 図面管理と検査管理が別部署で管理されており、連携ルールが定義されていなかったため

出力形式:
・「事象」→「なぜ1」→「なぜ5（根本原因）」の論理的つながりを確認し、コメント
・論理に飛躍や矛盾があれば指摘する（なければ「論理的に一貫しています」と明記）
・根本原因の特定: 「なぜ○にある○○が根本原因と判断されます」でまとめる
・根本原因に対応した恒久対策の例を2〜3個提案する（現場の確認・採否は担当者が判断）
```

## 暫定処置の報告文を作るプロンプト

クレーム発生直後に取引先へ送る「暫定処置報告」の文章を整理します。

```
以下の情報をもとに、取引先に送る「暫定処置報告」の文章を作成してください。

【基本情報】
不良内容: （例：外観不良（キズ）が受入検査で○個発見された）
発見日: （例：2026年7月20日）
取引先担当部署: （例：○○株式会社 品質保証部 ○○様）
自社担当: （例：品質保証部 ○○）

【暫定処置の内容】
- （例：(1) 御社で受入した当ロット（LOT-20260715-001）の在庫品を全数
  引き取り、自社にて全数再検査を実施いたします
  (2) 当ロット品は原則として御社ラインへの投入を停止いただけますでしょうか
  (3) 代替品を○月○日までに手配いたします）

【根本原因・恒久対策の調査状況】
（例：現在調査中。○月○日を目安に詳細報告書をご提出予定。）

出力形式:
・取引先への正式なビジネスメール形式
・最初に「ご迷惑とご心配をおかけしており、大変申し訳ございません」等の謝罪
・暫定処置の内容を明確に（番号付き）
・「詳細報告書は○月○日にご提出予定です」と締める
・結語・署名欄のプレースホルダー
```

## D1チームアサインのフォーマットを作るプロンプト

```
以下のメンバーで、8D対応チームのアサイン表を作成してください。

【対応チームメンバー】
- チームリーダー: ○○（品質保証部 主任）
- 製造担当: ○○（製造2課 班長）
- 生産技術担当: ○○（生産技術部）
- 品質検査担当: ○○（品質保証部）

【問題の概要】
（例：2026年7月15日、○○社より受入不良の連絡。出荷品のバリ混入）

出力形式:
・8D D1フォーム形式で
・チームメンバー氏名・所属・役割・連絡先（プレースホルダー）
・対応開始日・目標完了日の記入欄
```
