製造業のDX推進において、適切なITシステムの選定は重要な意思決定です。AIを使って要件定義・RFP（提案依頼書）・ベンダー比較表・導入スケジュールを効率的に整理できます。

---

## 生産管理システムの要件定義を整理するプロンプト

どんなシステムが必要かを明確にします。

```
以下の情報をもとに、生産管理システムの
要件定義書を整理してください。

【当社の現状と課題】
業種: 精密板金加工（受託製造）/ 従業員: 45名

現在のIT環境:
  ・受注・出荷管理: Excelで管理（担当者によってフォーマットが異なる）
  ・在庫管理: 紙台帳＋週1回の手入力
  ・生産計画: ホワイトボード＋Excelスケジュール
  ・原価計算: 経理ソフトに一部連動しているが手入力多い

現在の主な課題:
  ①受注→生産計画→在庫の情報が連動しておらず、
    二重入力・転記ミスが週5〜10件発生
  ②在庫の実数が把握できておらず、欠品や過剰在庫が頻発
  ③部門間の情報共有が遅く、「聞かないと分からない」状態
  ④原価の把握が月次決算まで分からない

【導入したいシステムの方向性】
  ・クラウド型（初期投資を抑えたい）
  ・月額費用の目安: ¥100,000以下/月
  ・スマートフォンでも操作できること（現場作業者用）
  ・会計ソフト（弥生会計）との連携が必要

出力形式:
・「システム要件定義書の構成と各項目の記載内容」
・「機能要件の優先順位（Must Have / Should Have / Nice to Have）」
・「非機能要件（セキュリティ・可用性・スケーラビリティの基準）」
・「現在のExcel管理からの移行要件
  （既存データの移行・既存業務フローとの整合性）」
・「費用対効果の試算（システム導入で削減できる工数×人件費）」
```

## ベンダー選定のためのRFPを整理するプロンプト

複数のベンダーに同じ条件で提案させます。

```
以下の情報をもとに、生産管理システムの
RFP（提案依頼書）を整理してください。

【RFPの発行対象】
  候補ベンダー3社への同時発送を予定
  回答期限: 発行後4週間
  提案プレゼン: 3社それぞれ1時間

【RFPに含める主な内容】
会社概要と現状:
  業種・規模・現状の課題（上記要件定義より）

求める提案内容:
  ①機能一覧への対応可否（○・△・×で回答）
  ②導入スケジュール（契約から稼働まで）
  ③初期費用・月額費用の明示（カスタマイズ費用も含む）
  ④実績（同業種・同規模の導入事例）
  ⑤サポート体制（導入後のヘルプデスク・オンサイト対応）
  ⑥トレーニング内容（操作研修・マニュアル提供）

評価基準（重み付け）:
  機能適合性: 30% / 費用: 25% / 実績・信頼性: 20%
  サポート: 15% / 使いやすさ: 10%

出力形式:
・「RFP文書の構成と各項目の記載内容（A4・10〜15ページ）」
・「機能要件チェックシートのフォーマット
  （機能名・当社の要件・優先度・ベンダー回答欄）」
・「ベンダー比較評価表のフォーマット
  （3社×評価項目×重み付けスコアの一覧表）」
・「プレゼン当日の質問リスト（10問）」
  （例: 「カスタマイズの上限費用は？」「データの移行は誰がやるか？」）
・「最終的なベンダー選定の決裁文書の骨子」
```

## システム導入プロジェクト計画書を整理するプロンプト

ベンダー決定後の導入を成功させる計画を立てます。

```
以下の情報をもとに、生産管理システムの
導入プロジェクト計画書を整理してください。

【決定した内容】
採用システム: クラウド型生産管理システム「FACTORY-X」（仮）
採用ベンダー: ○○システムソリューションズ
契約日: 2026年7月1日
目標稼働日: 2026年10月1日（3ヶ月後）
費用: 初期¥1,200,000 / 月額¥85,000

プロジェクト体制:
  プロジェクトオーナー: 工場長・田中
  プロジェクトマネージャー（社内）: 管理部・佐藤
  ベンダー側PM: 鈴木（○○社担当）
  利用ユーザー: 管理部3名・製造部班長5名・品質部2名

導入の懸念点:
  ・現場作業者（40代以上多い）のシステム習熟に不安
  ・稼働初日に受注対応が滞らないか
  ・現在のExcelデータ（3年分）の移行が大変そう

出力形式:
・「導入プロジェクトのWBS（主要タスク・担当・期限の一覧）」
  フェーズ1（7月）: 要件確認・設定・データ移行準備
  フェーズ2（8月）: テスト・ユーザー研修
  フェーズ3（9月）: 並行運用（新旧システム同時稼働）
  フェーズ4（10月）: 本番稼働
・「データ移行計画（どのExcelデータを・誰が・どのように移行するか）」
・「ユーザー研修計画（対象者別・研修内容・時間配分）」
・「稼働後のリスク対応計画（トラブル時の連絡体制・切り戻し手順）」
・「プロジェクト定例会議の運用（週次進捗確認のアジェンダ）」
```
