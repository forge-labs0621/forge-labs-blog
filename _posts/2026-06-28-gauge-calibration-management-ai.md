ISO 9001では測定器・ゲージ・計量器の校正管理が要求されています（7.1.5条）。内部監査でも確認されやすい項目であり、校正台帳・校正証明書の管理・期限切れ防止の仕組みをAIで整理できます。

---

## 測定器・ゲージ管理台帳のフォーマットを作るプロンプト

工場全体の測定器を一覧管理する台帳のフォーマットを作ります。

```
以下の情報をもとに、測定器・ゲージ管理台帳のフォーマットと
現有機器リストの初期登録案を作成してください。

【自社で使用している測定器・ゲージ（例）】
① ノギス（デジタル）: 5本 / 外部校正（校正業者）/ 年1回
② マイクロメーター（外径）: 3本 / 外部校正 / 年1回
③ ハイトゲージ（デジタル）: 2台 / 外部校正 / 年1回
④ 三次元測定機（CMM）: 1台 / メーカー校正 / 年1回
⑤ 穴径ゲージ（限界ゲージ）: 10本 / 社内比較校正 / 6か月ごと
⑥ 硬さ計（ロックウェル）: 1台 / 外部校正 / 年1回
⑦ 台ばかり（1ton用）: 2台 / 検定（計量法） / 年1回

【管理に含めたい項目】
・管理番号（シリアル番号）
・機器名・型式・メーカー
・使用工程・保管場所
・校正の種類（外部校正・社内比較・計量法検定）
・校正周期
・最終校正日・次回校正期限
・校正状況（有効 / 期限切れ / 修理中）
・担当者

出力形式:
・測定器・ゲージ管理台帳のフォーマット（表形式）
・上記7種類の機器の登録例（管理番号の付け方含む）
・「校正期限が近い機器（1か月以内）を自動でアラートする方法」の提案
・ISO 9001 7.1.5条への対応確認チェックリスト
```

## 社内比較校正の手順書を作るプロンプト

限界ゲージなど、社内で比較校正を行う場合の手順を整理します。

```
以下の情報をもとに、社内比較校正の手順書を作成してください。

【対象機器と校正の概要】
機器名: （例：穴径限界ゲージ（φ10H7用 通り側・止まり側））
校正の目的: 校正機関の校正済みマスターゲージと比較し、
  現場使用ゲージの偏差（消耗）を確認する

【校正の手順（現在の方法）】
1. 校正済みマスターゲージ（外部校正・有効期限内）を準備する
2. 測定環境: 室温20±2℃の測定室 / 測定前に15分以上順化
3. 比較測定: 品質保証担当者が現場ゲージで5回測定 → 平均値を算出
4. 合否判定: マスター値との差異が ±0.005mm以内 → 合格（使用継続）
   ±0.005mmを超える → 不合格（使用禁止・廃棄または修理）
5. 記録: 社内校正記録簿に記録（測定者・日付・結果）

確認してほしいこと:
・この手順でISO 9001の社内比較校正の要件を満たしているか
・手順書として不足している記載項目はないか
・「不合格となった場合の処置（使用禁止の識別方法等）」の追加

出力形式:
・社内比較校正手順書のフォーマット（STEP形式）
・ISO 9001 7.1.5.1b の「自分で定めた基準による定期的な校正」への対応確認
・社内校正記録簿のフォーマット（日付・対象ゲージ管理番号・測定者・結果・合否）
```

## 校正期限切れ発見時の対応記録を作るプロンプト

内部監査等で校正期限切れが見つかった場合の是正処置記録を作ります。

```
以下の情報をもとに、測定器の校正期限切れに関する
是正措置記録を作成してください。

【不適合の内容】
発見日・状況: （例：2026年10月5日 内部監査中に発見。
  製造部のノギス3本（管理No. NG-02, NG-03, NG-07）の校正期限が
  2026年9月末に切れていたが、そのまま現場で使用されていた）

【原因分析】
直接原因: （例：校正台帳の確認を毎月末に担当者が行うことになっていたが、
  9月末の確認が実施されなかった）
根本原因: （例：校正期限の確認が個人の記憶・カレンダーに依存しており、
  確認の実施を確認する仕組みがなかった）

【是正処置】
即時対応（実施済み）:
  （例：3本のノギスを即時使用禁止・隔離。「使用禁止」タグを貼付。
  校正業者に翌営業日の校正を依頼）
再発防止:
  （例：校正台帳をExcelからGoogleスプレッドシートに移行し、
  期限1か月前に担当者と品質保証部長に自動メール通知を設定する。
  毎月1日の品質会議で台帳の期限確認を議題に固定する）
影響評価:
  （例：期限切れ期間（2026年10月1日〜10月5日）に加工した製品の
  影響確認を実施。測定値は基準内で問題なしと判断。出荷済み品は
  不適合品として処理しない（影響ないと評価））

出力形式:
・是正措置報告書（不適合番号 / 発見状況 / 原因 / 処置 / 効果確認計画）
・「影響評価結果」の記録
・「確認完了サイン欄」（担当者・品質保証部長・是正後フォローアップ）
```
