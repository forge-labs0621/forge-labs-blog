緊急受注への対応は顧客満足を高める一方、計画外のコストと混乱を招きます。AIを使って特急対応の可否判断・生産計画変更・費用確認・顧客対応書類を体系化できます。

---

## 緊急受注・特急対応の管理書類を整理するプロンプト

```
以下の情報をもとに、製造業の緊急受注・特急対応時の
管理書類と対応フローを整理してください。

【緊急受注の状況】
顧客からの要請: 「既存発注品を通常纳期（2週間後）より1週間早めてほしい」
                あるいは「急ぎで追加100個 欲しい（明後日まで）」

特急対応の際に発生するコスト・影響:
  ①他の受注品の生産を後回しにする（他顧客に影響）
  ②残業・休日出勤が必要になる場合がある
  ③資材の緊急手配コストが発生する
  ④製造現場の段取り変更による効率低下

特急対応の社内フロー（理想）:
  ①受注受付 → ②製造・資材への可否確認（1時間以内）
  → ③対応可否と追加費用の判断 → ④顧客への回答
  → ⑤生産計画変更の指示 → ⑥出荷・完了報告

出力形式:
・「特急受注受付票のフォーマット
  （要請日 / 要請内容 / 希望纳期 / 現在の納期 / 製造確認結果 / 対応可否）」
・「特急対応可否の社内確認チェックリスト（在庫 / 設備空き / 人員 / 資材）」
・「顧客への特急対応回答メール文例
  （対応可能な場合 / 困難な場合 / 一部対応できる場合 の3パターン）」
・「特急費用の請求根拠書（残業費・資材緊急調達費・段取りコスト等の根拠）」
・「特急受注が頻発する根本原因の分析フレームと対策（受注計画・リードタイム短縮）」
```
