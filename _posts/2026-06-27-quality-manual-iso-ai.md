ISO 9001（品質マネジメントシステム）の取得・維持・更新に必要な品質マニュアル・手順書・記録類は、文書量が多く整備・更新の負担が大きい書類です。AIを使うと、品質方針の文章化・手順書の骨子作成・内部監査チェックリストの準備を効率化できます。

> **重要**: AIは品質マニュアルの「下書き・骨子作成・文章整理」を補助します。ISO 9001審査に使用する文書の最終確認・認証審査対応は、必ず品質管理責任者または認証機関が行ってください。

## AIが役立つISO 9001文書管理の場面

| 場面 | AIでできること |
|---|---|
| 品質方針の文章化 | 経営者の意図をもとに品質方針の文面を整理 |
| 手順書の骨子作成 | 口頭で説明していた手順をドキュメント化 |
| 内部監査チェックリスト | 各条項に対応した確認項目の一覧を作成 |
| 不適合管理の記録 | 不適合・是正処置の記録を整理する |
| 文書の見直しコメント | 既存文書の改訂箇所の洗い出しを補助 |

## 品質方針の文章を整理するプロンプト

経営者の言葉をもとに、ISO 9001に適した「品質方針」の文章を作ります。

```
以下の経営者の考え・方針をもとに、ISO 9001に沿った品質方針の
文章を作成してください。

【会社・事業の概要】
（例：自動車部品の受託加工業。従業員30名。プレス加工・溶接・表面処理を
行う。主な取引先は自動車部品メーカーおよびTier1サプライヤー）

【経営者が大切にしていること（話し言葉で構いません）】
（例：
・品質は全員参加で守るもの。担当者だけでなく全員が品質に責任を持つ
・顧客に迷惑をかけない、不良を絶対に流出させない
・現場で気づいた改善点はすぐに提案して改善する文化をつくる
・ISO取得は目標ではなく、品質を継続的に向上させる手段
）

出力形式:
・ISO 9001の品質方針として公文書で使える形式
・200字以内でコンパクトに
・「顧客満足」「継続的改善」「全員参加」の3要素を含めること（ISO 9001の要求）
・版番号・制定日・承認者欄（プレースホルダー）付き
```

## 手順書の骨子を口頭説明からドキュメント化するプロンプト

「やり方は口頭で伝えている」「ベテランの頭の中にしかない」手順を文書化するための骨子を作ります。

```
以下の口頭説明・メモをもとに、ISO 9001の「手順書」として使えるように
文書の骨子を作成してください。

【手順書にしたい業務・作業】
（例：溶接後の外観検査手順）

【現在の口頭説明・やり方のメモ】
（例：
・溶接品が前工程から来たら、まずビードの外観を目視で確認する
・確認するのは、ブローホール（気孔）・割れ・アンダーカット・スパッタの付着
・不合格品は赤いタグをつけて「不適合品置き場」に移す
・合格品は緑のタグをつけて次工程へ送る
・記録は「溶接検査記録シート」に日付・ロット番号・検査員名・
  OK/NG数・NG内容を記入する
・月末に記録シートを品質担当に提出する
）

出力形式:
・「外観検査手順書」（または適切なタイトル）として
・目的・適用範囲
・必要な設備・工具・記録様式の一覧
・作業手順（番号付きステップ形式）
・合否判定基準の概要
・記録・保管方法
・版番号・制定日・承認者欄（プレースホルダー）
```

## 内部監査チェックリストの骨子を作るプロンプト

ISO 9001の内部監査で使うチェックリストの骨子を作ります。

```
以下の条件で、ISO 9001内部監査のチェックリストを作成してください。

【監査対象の部門・プロセス】
（例：製造部門（加工・組立工程）)

【重点的に確認したい事項（複数選択可）】
□ 品質計画・目標の管理（品質目標が現場に展開されているか）
□ 作業手順書・仕様書の管理（最新版が現場にあるか）
□ 設備の管理（点検記録・校正記録が適切か）
□ 不適合品の管理（不適合品の識別・隔離・処置が適切か）
□ 記録の管理（記録が適切に保管・整理されているか）
□ 是正処置の有効性確認（前回の指摘事項が改善されているか）

出力形式:
・「ISO 9001内部監査 チェックリスト（製造部門）」として
・各項目を「確認事項 / 確認方法 / 証拠・記録 / OK/NG/観察」の列で表形式に
・重大不適合・軽微不適合・観察事項の判定基準の注記を末尾に
・監査日・監査員・被監査部門・承認者欄
```

## 不適合・是正処置報告書を整理するプロンプト

内部監査・クレーム・工程内不良に対して是正処置を記録する書類です。

```
以下の情報をもとに、ISO 9001の「不適合・是正処置報告書（CAR）」を
作成してください。

【不適合の内容】
発生日: （例：2026年7月15日）
発生場所・工程: （例：製造部門 溶接後外観検査工程）
不適合の説明: （例：外観検査を通過した溶接品に、後工程で小さなブローホールが
3個発見された。計10個のロットのうち3個が不合格。）

【暫定処置（すでに行ったこと）】
（例：当該ロット全数を不適合品置き場に移動し、全数再検査を実施した。
残り7個は合格品と確認。不合格3個は溶接やり直し処置を行う。）

【原因分析（わかっている範囲で）】
（例：
・外観検査の照明が不十分で小さな欠陥を見落とした可能性が高い
・検査員が当日の午後から別業務と兼務しており、注意力が低下していた可能性
）

【是正処置（再発防止策）】
（例：
・検査台の照明をLED照明に交換し、照度を2倍に改善する（7/20まで）
・外観検査は専任担当で実施し、兼務体制を解消する（8月から）
・検査基準書に「照明条件」を明記し、点検チェックリストを更新する（7/25まで）
）

出力形式:
・「不適合・是正処置報告書」として
・不適合の内容・影響範囲
・暫定処置（封じ込め）
・原因分析（なぜなぜ分析の形式で整理してください）
・是正処置（再発防止策・担当者・期限）
・有効性確認の予定（処置後の確認時期・方法）
・報告者・承認者・有効性確認者欄
```

## 既存の品質文書を見直すためのプロンプト

古い文書や表現が曖昧な手順書を整理する際に使えます。

```
以下の手順書の一部をレビューし、改善すべき点を指摘してください。

【現在の手順書の内容（貼り付けてください）】
（ここに現在の手順書のテキストを貼り付ける）

【確認してほしい観点】
□ 手順の抜け漏れがないか
□ 判断基準が曖昧な表現がないか（例:「十分に」「適宜」などは具体化を推奨）
□ 作業者が初めて読んだときに理解できるか
□ ISO 9001の「手順書に求められる要素（目的・適用範囲・手順・記録）」が含まれているか

出力形式:
・改善すべき点を箇条書きで指摘
・各指摘に対して「改善案の例」を一文添える
・文書全体の評価（良い点・改善余地のある点）をまとめとして記載
```

## AIを使う際の注意

- ISO 9001の認証審査は認定審査機関（登録審査機関）が行います。AIが作成した文書がISO 9001の要求事項を満たしているかどうかの最終判断は、必ず品質マネジメント担当者または認証機関に確認してください
- 品質マニュアル・手順書は会社の品質管理体制を定義する重要文書です。AIが作成した下書きを確認せずにそのまま使用することは避けてください
- ISO 9001文書には「版管理」が必要です。作成後は版番号・改訂履歴の管理を適切に行ってください
