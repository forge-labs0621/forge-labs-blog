製造業の経理・営業担当者にとって、毎月の請求書発行と送付は定型作業でも確認事項が多い書類です。金額・品目・振込先の記載ミスは取引先とのトラブルにつながるため、AIを活用して作成の手間を減らしつつ、ミスを防ぐことが重要です。

## 請求書と納品書の違い

| | 納品書 | 請求書 |
|---|---|---|
| タイミング | 商品・サービスの納品時 | 支払いを請求するとき |
| 主な内容 | 納品した品物・数量・単価 | 請求金額・振込先・支払期日 |
| 法的位置づけ | 取引の証明 | 支払義務の通知 |

製造業では「納品書と請求書を同封する」「月末に請求書をまとめて送る」など取引先によって異なることが多いです。

## 請求書の送付状（カバーレター）を作るプロンプト

請求書本文（金額・品目）は会計ソフトで作成し、添付の送付状・メール文面をAIで作れます。

---

```
以下の情報をもとに、請求書の送付状（添え状）を作成してください。

【送付先情報】
取引先名: （プレースホルダー可）
担当者名: （プレースホルダー可、または「ご担当者様」）

【請求内容の概要】
（例：
・2026年6月分の加工代金として、請求書1通を同封します
・請求金額: 別紙請求書の通り
・お支払期日: 2026年7月31日
）

【特記事項】
（例：なし / または「今月から振込先口座が変更になりました」など）

出力形式:
・会社から取引先への送付状として
・日付・差出人・宛先・件名・本文・同封書類の一覧
・文体はビジネス標準（丁寧だが簡潔に）
・金額は「別紙請求書の通り」とし、送付状には具体的な金額を記載しない
```

## 請求書のメール送付文を作るプロンプト

請求書をメールに添付して送る場合の本文もAIで作れます。

```
以下の情報をもとに、請求書を添付したメールの本文を作成してください。

【送付先】
取引先名・担当者名: （プレースホルダー可）

【メールの状況】
（例：
・毎月定例の請求書送付（月末に送るパターン）
・初めての請求書送付（新規取引先）
・追加請求が発生したので今月2回目の送付
）

【請求内容の概要】
（例：2026年6月分加工費 請求書1通（ファイル名: 請求書_2606_○○様.pdf）を
添付します。お支払期日: 2026年7月末日）

【特記事項】
（例：なし / または「振込先口座が変更になりましたので添付のご確認をお願いします」）

文体: ビジネスメール・丁寧に・要点を先に書く形式
```

## 振込先変更の案内文を作るプロンプト

銀行口座が変わったときの案内文も素早く作れます。

```
以下の内容で、取引先への振込先変更案内文を作成してください。

【変更内容】
旧振込先: （プレースホルダー / 文面に「旧口座」と表記）
新振込先: （プレースホルダー / 文面に「新口座」と表記）
変更適用日: （例：2026年8月1日お支払い分から）

【理由（記載するかどうか）】
□ 記載する（例：メインバンクの変更のため）
□ 記載しない（一般的）

注意点: 振込詐欺・フィッシング対策のため、メールで口座番号を送る際は
「電話での確認をお願いする」文言を追加してください。

出力形式:
・お知らせ文書として整形
・電話確認の依頼文言を必ず含める
・担当者連絡先（プレースホルダー）を明記
```

## 入金確認の催促メールを作るプロンプト

支払期日を過ぎても入金がない場合の催促メールも、AIで丁寧な表現に整えられます。

```
以下の状況で、入金催促メールを作成してください。

【状況】
（例：
・お支払い期日（7月31日）を1週間過ぎても入金が確認できていない
・これが初めての催促（関係が良好な取引先）
・または、2回目の催促（前回も同様の遅延があった）
）

【催促する金額・請求書番号】
（例：¥○○○（税込）/ 請求書No.2607-○○○）

【相手への配慮】
（例：
・「入違いの場合はご容赦ください」という文言を入れてほしい
・「関係を傷つけたくない、ただ確認したい」というトーン
）

文体:
・丁寧・ビジネス標準
・責め立てる表現にならないよう注意
・確認と次のアクション（連絡先）を明確に書く
```

## 見積書・納品書・請求書の番号管理ルールを整理するプロンプト

書類番号の採番ルールが決まっていない場合、AIと一緒に決められます。

```
以下の条件で、製造業向けの書類番号（ドキュメント番号）の採番ルールを
設計してください。

【管理したい書類の種類】
（例：見積書・注文書・納品書・請求書の4種類）

【現在の状況】
（例：
・書類番号がバラバラで管理できていない
・エクセルで管理しているが、重複や抜けが出る
・取引先ごとに番号を振りたい
）

【希望する条件】
（例：
・年度・月・連番を含めたい（例: 見積2607-001）
・取引先コードを含めたい（例: 見積-ABC001-001）
・シンプルに管理したい
）

出力形式:
・推奨する採番ルールを2〜3案提示
・各案の長所・短所
・運用上の注意点（重複防止・欠番の扱いなど）
```

---

## AIを使う際の注意

- **請求金額・振込先は必ず自分で確認する**: AIは入力した内容を整形するだけです。金額の計算・振込先の正確性は必ず人が確認してください
- **請求書の原本（会計ソフト・Excelの書式）にはAIを使わない**: 法的に有効な請求書の書式・税務処理は会計ソフトまたは専門家が担当してください
- **振込先情報はメールだけで送らない**: フィッシング詐欺対策として、振込先変更は電話でも確認することをお勧めします
