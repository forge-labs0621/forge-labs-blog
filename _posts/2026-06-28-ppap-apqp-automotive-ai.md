自動車部品サプライヤーでは、新規部品の量産開始前に客先への提出が求められるPPAP（生産部品承認プロセス）書類を準備する必要があります。FMEA・コントロールプラン・初期工程性能評価などの書類作成をAIで補助できます。

---

## PFMEA（工程FMEA）の作成補助プロンプト

潜在的な工程不良の可能性と対策を整理するPFMEAを作ります。

```
以下の情報をもとに、PFMEA（工程故障モード影響解析）の
骨子を作成してください。

【製品・工程の概要】
品番・品名: （例：A-1234 アルミダイカスト ハウジング）
対象工程: （例：CNC機械加工（穴あけ・面加工））
潜在的なリスク（事前にわかっている懸念点）:
  （例：
  ・穴径のばらつき（工具摩耗による）
  ・面粗さの悪化（切削速度・送り量の設定ミス）
  ・切り粉の巻き込み（洗浄不足による内部残留）
  ）

【FMEA分析の対象】
（例: 穴あけ工程について分析してほしい）
工程の目的: （例: φ10H7の精密穴を加工する）
設計要求・特殊特性: （例: 穴径公差 φ10+0.015/-0.000 / 表面粗さ Ra1.6以下）

出力形式:
・PFMEA表形式（工程機能 / 故障モード / 影響 / 重要度S /
  原因 / 頻度O / 現行の検出管理 / 検出度D / RPN /
  推奨処置 / 責任者・期限）
・RPN = S × O × D の計算（各1〜10で評価）
・「優先度が高い故障モード（RPN 100以上等）」への推奨処置
・FMEAの更新タイミングの説明（4M変更時・クレーム発生時等）
```

## PPAPサブミッション準備のチェックリストを作るプロンプト

客先へのPPAP書類提出前の準備チェックリストを作ります。

```
以下の情報をもとに、PPAPサブミッションに向けた
準備チェックリストを作成してください。

【自社の状況】
取引先・提出要求レベル: （例：○○自動車工業 / PPAP レベル3
  （客先への書類提出が必要））
新規部品: （例：A-1234 アルミダイカストハウジング / 新車種向け新部品）
PPAP提出期限: （例：2026年8月15日）

【PPAP 18要素の準備状況確認】
（AIAG PPAP 4th editionの主要要素）
1. 設計記録（図面）: 客先支給の最新図面（Rev.C）を入手済み
2. 承認された設計変更書類: 対象外（新規品）
3. 技術承認（顧客技術部署の承認）: 未取得（提出後に取得予定）
4. DFMEA（設計FMEA）: 客先が実施（自社には設計権限なし）
5. 工程フロー図: 作成済み
6. PFMEA（工程FMEA）: 作成中（80%完成）
7. コントロールプラン: 作成中
8. MSA（測定システム解析）: ゲージR&Rを実施予定
9. 寸法結果: 図面全点の寸法測定 → 30個の測定完了
10. 材料・性能試験結果記録: 材料成績書（仕入先から取得済み）
...

出力形式:
・PPAP 18要素のチェックリスト（完了 / 未完了 / 対象外 / 顧客確認待ち）
・各要素の「残作業と担当者・期限」を記入する欄
・PPAP提出前の「最終確認事項」リスト（承認者サイン・提出方法確認等）
・「レベル3の場合に物理的に提出が必要な書類」と「自社保管でよい書類」の区分
```

## APQP進捗報告書を作るプロンプト

新製品立上げのAPQP（先行製品品質計画）の進捗を客先に報告します。

```
以下の情報をもとに、客先向けのAPQP進捗報告書を作成してください。

【新製品・プロジェクトの概要】
品番・品名: （例：B-9999 ステアリング支持ブラケット）
量産開始予定: （例：2027年1月 SOP（Start of Production））
PPAPサブミッション期限: （例：2026年10月）
客先の担当窓口: （例：○○自動車工業 SQE ○○様）

【APQPフェーズ別の進捗（現時点）】
フェーズ1（計画・設計インプット）: 完了 ✓
  ・客先仕様書受領 / 特殊特性の識別 / 製造可能性評価 完了
フェーズ2（製品設計・検証）: 完了 ✓
  ・DFMEAは客先実施 / 自社での加工性評価完了
フェーズ3（工程設計・検証）: 進行中（65%）
  ・工程フロー図 完了 / PFMEA 作成中（80%）/
  コントロールプラン 未着手 / 設備・治具の製作 完了
フェーズ4（製品・工程妥当性確認）: 未着手
  ・試作ロット（30個）の加工予定: 2026年7月
フェーズ5（フィードバック・評価）: 未着手

出力形式:
・客先報告用のAPQP進捗報告書（表形式）
  （フェーズ / 主要アクション / 状態（完了/進行中/未着手）/ 予定日 / 担当者）
・「遅延リスクがある項目」とその理由・対応策
・「次回の進捗報告予定日」の提案
```
