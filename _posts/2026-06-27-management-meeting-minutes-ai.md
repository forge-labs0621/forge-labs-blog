製造業では月次の経営会議・品質会議・部門会議の議事録作成が定例業務になっています。メモや会話の記録をもとに正式な議事録にまとめる作業は手間がかかりますが、AIを使うと素早く整理できます。

> この記事では「経営会議・品質会議・部門会議」の議事録を扱います。朝礼・申し送りの議事録については[別記事](/forge-labs-blog/2026/06/21/morning-meeting-ai.html)をご参照ください。

## AIが役立つ会議議事録作成の場面

| 会議の種類 | 議事録に必要な要素 |
|---|---|
| 月次経営会議 | 売上・受注状況、経費・予算実績、課題と対策、次月の方針 |
| 品質会議（QC会議） | 不良率・クレーム件数、原因と対策、改善目標の進捗 |
| 安全衛生委員会 → [別記事参照](/forge-labs-blog/2026/06/27/safety-committee-minutes-ai.html) | |
| 部門長会議 | 各部門の進捗・課題・連絡事項 |
| 取引先との定例会議 | 受注状況、品質課題、今後のスケジュール |

## 月次経営会議の議事録を作るプロンプト

```
以下の会議メモをもとに、月次経営会議の議事録を作成してください。

【会議の基本情報】
日時: （例：2026年7月10日（木）15:00〜17:00）
場所: （例：本社 会議室）
出席者: （例：社長・工場長・営業部長・製造部長・経理部長 計5名）
欠席者: （例：なし）
議事録担当: （例：総務 山田○○）

【会議の主な議題と話し合われた内容（メモ）】
（例：
1. 6月度 売上・受注報告（営業部長報告）
   ・6月売上: ¥32,500,000（予算比 103%、前年比 98%）
   ・受注残高: ¥85,000,000（前月比 +3,000,000）
   ・新規引合い2件（自動車部品系1件、電子機器系1件）

2. 6月度 生産・品質報告（製造部長報告）
   ・稼働率: 91%（目標95%。設備故障による2日間停止が影響）
   ・不良率: 1.2%（目標1.0%以下、改善継続中）
   ・7月の生産計画: 設備修繕完了後、フル稼働予定

3. 課題・対策
   ・旋盤1台の故障対応: 8月末に修繕完了見込み。それまでは外注で補完
   ・新規取引先への対応: 8月中にサンプル提出予定

4. 次月の方針
   ・7月売上目標: ¥33,000,000
   ・品質目標: 不良率1.0%以下を必達
   ・次回会議: 8月7日（木）15:00〜
）

出力形式:
・正式な「月次経営会議 議事録」として
・各議題を「報告・審議・決定事項」の形式で整理
・アクションアイテム（誰が・何を・いつまでに）を最後に表でまとめる
・議事録担当・確認者（議長）・日付欄
```

## 品質会議（QC会議）の議事録を作るプロンプト

月次や週次で行う品質管理会議の議事録です。

```
以下のメモをもとに、品質会議の議事録を作成してください。

【会議の基本情報】
日時: （例：2026年7月3日（金）14:00〜15:30）
出席者: （例：品質部長・製造課長・設計担当・検査担当 計4名）

【議題と内容（メモ）】
（例：
1. 6月度 品質実績報告
   ・社内不良率: 1.8%（目標1.0%、依然高い水準）
   ・主な不良内容: 溶接ブローホール 60件、寸法外れ 35件
   ・顧客クレーム: 0件（3ヶ月連続ゼロ達成）

2. 不良件数の多いブローホール問題の原因分析
   ・なぜなぜ分析結果: 
     溶接条件の管理が担当者の判断に依存していた
     → 溶接条件表の見直しと、条件設定の手順化が必要
   ・対策: 溶接条件を作業手順書に明記する（7月末まで）

3. 前月の是正処置の効果確認
   ・寸法外れ対策（3月実施）: 件数が70件→35件に半減。効果あり。継続。

4. 来月の品質目標と重点活動
   ・不良率目標: 1.5%以下（ステップ目標として）
   ・重点活動: 溶接条件の手順書化・全員周知
）

出力形式:
・「品質会議 議事録」として
・品質実績を数値で整理（前月比・目標との比較を明示）
・是正処置・アクションアイテムを「担当者 / 内容 / 完了期限」の表で
・次回会議予定
・議事録作成者・確認者欄
```

## 取引先との定例会議の記録を作るプロンプト

顧客・仕入先との定例打ち合わせの議事録・覚書として使えます。

```
以下の打ち合わせメモをもとに、取引先定例会議の記録（議事録）を
作成してください。

【会議の基本情報】
日時・場所: （例：2026年7月5日（火）10:00〜11:30 / 先方訪問）
出席者:
  ・自社: （例：営業部長 田中、品質担当 鈴木）
  ・先方: （例：株式会社○○製作所 資材課 山田様、品質保証課 佐藤様）
会議の目的: （例：月次定例打ち合わせ（品質・受注状況確認））

【打ち合わせの内容（メモ）】
（例：
・受注確認: 7月分 200個の受注内示を受領。単価・仕様に変更なし。
・品質状況: 6月の不良流出ゼロを先方より評価いただいた。
  先方から「今後は月200個→300個への増量を検討したい」との話あり。
・仕様変更の連絡: 8月以降、表面処理の指定が変わる可能性あり。
  詳細は先方設計部門で決定後、別途連絡をいただける。
・次回打ち合わせ: 8月5日（火）同時刻・先方訪問
）

出力形式:
・「定例会議記録・覚書」として
・決定事項と確認事項を区別して記載
・自社・先方それぞれのアクションアイテムを表で整理
・「この議事録の内容に相違があれば○日以内にご連絡ください」の一文
・作成日・作成者・配布先
```

## 長い会議メモを整理するプロンプト（汎用）

どんな会議でも使える汎用の整理プロンプトです。

```
以下の会議メモを整理して、正式な議事録を作成してください。

【会議メモ（そのまま貼り付けてください）】
（会議中に取ったメモ、音声起こしのテキスト、箇条書きのメモなど）

【会議の基本情報】
・会議名・目的:
・日時・場所:
・出席者:

整理のルール:
・「決定事項」「保留事項・継続検討事項」「アクションアイテム（担当者・期限）」を分けて整理する
・具体的な数値や固有名詞はメモの通りに記載し、勝手に変えない
・曖昧な部分は「（要確認）」と記載する
・メモに登場した誰かが「〇〇します」「検討します」「持ち帰ります」と言った場合、アクションアイテムとして抽出する
```

## AIを使う際の注意

- 議事録は「発言の記録」です。AIが「整理・要約」した結果と、実際の発言・決定事項が変わっていないか、必ず会議担当者が確認してください
- 取引先との会議記録は、後のトラブル防止のためにも正確性が重要です。送付前に出席者が内容を確認してください
- 決定事項・アクションアイテムが正確に記録されているかが最も重要です。要約や文体の整え方はAIに任せ、事実の確認は担当者が行う、という役割分担が推奨です
