## DX推進計画書の骨子を作るプロンプト

工場・製造業のDX推進計画書（経営層・補助金申請向け）を作ります。

```
以下の情報をもとに、製造業のDX推進計画書の骨子を作成してください。

【自社の現状】
会社規模・業種: （例：金属プレス加工業 / 従業員65名）
現在のIT活用状況:
  （例：会計はマネーフォワードを使用。生産管理は手書きの日報と
  Excelが中心。設備点検記録は紙。受発注はFAXと電話が主体）
DX化を検討しているきっかけ:
  （例：ベテランの退職で技術・データが属人化している。
  ものづくり補助金のDX加点を狙いたい。
  取引先から電子受注対応を求められている）

【DXで解決したい課題（上位3つ）】
（例：
1. 生産実績データの集計に毎月2〜3日かかっている（手入力による転記）
2. 現場の作業日報が紙のため、集計・分析できていない
3. 設備トラブルの履歴がバラバラで、予防保全の計画が立てられない
）

【検討している解決策】
（例：
1. 生産管理システム（クラウド型）の導入
2. IoTセンサーによる設備稼働データの自動収集
3. タブレット端末で現場日報・点検記録を入力
）

出力形式:
・DX推進計画書のフォーマット（現状分析→課題→解決策→期待効果→スケジュール）
・3年間のロードマップ（フェーズ分け）
・「費用対効果の試算」のプレースホルダー（数値を入れる箇所を示す）
・補助金申請でよく聞かれる「従業員のデジタルスキル向上策」の記載
```

## IT導入補助金申請書類の下書きを作るプロンプト

IT導入補助金の事業計画書（申請書）の下書きを作ります。

```
以下の情報をもとに、IT導入補助金の事業計画書の下書きを
作成してください。

【導入予定のITツール】
ツール名・カテゴリ: （例：クラウド型生産管理システム ○○（受注管理・工程管理・
  実績集計機能）/ ベンダー: ○○株式会社）
補助申請額: （例：導入費用 ¥250万円 / 補助率1/2 → 補助申請 ¥125万円）

【現状の課題（申請書に書く内容）】
（例：現在は受注情報をFAXで受取り手入力で転記しており、
入力ミスや集計に時間がかかっている。月次の生産実績レポートの
作成に担当者が丸2日かかっており、即時の現状把握ができていない）

【導入後の期待効果（数値目標）】
（例：
・受注データの入力時間: 月40時間 → 月10時間（▲75%削減）
・月次レポート作成: 2日 → 半日（▲75%削減）
・在庫管理の精度向上: 棚卸差異 2.1% → 0.5%以下目標
）

出力形式:
・IT導入補助金申請で記載が求められる「業務プロセスの変化」の説明
・「導入前（As-Is）」と「導入後（To-Be）」の対比（表形式）
・「労働生産性の向上（数値目標）」の記載
・自社の強み・市場での位置付けの記載（採点官に伝えるアピールポイント）
・※最終的な申請書は補助金事務局の記載要領に合わせること
```

## デジタル化の効果測定・報告書を作るプロンプト

IT・DXツール導入後の効果測定結果を経営層・補助金報告向けに整理します。

```
以下のデータをもとに、DX・IT導入の効果測定報告書を
作成してください。

【導入ツールと導入時期】
（例：クラウド型生産管理システム / 2026年4月から運用開始）

【効果測定結果（導入前後の比較）】
業務効率化:
  ・月次レポート作成時間: 導入前 16時間 → 導入後 4時間（▲75%）
  ・受注入力ミス: 月平均5件 → 0件（▲100%）
  ・在庫確認の問い合わせ電話: 週20件 → 週3件（▲85%）
コスト影響:
  ・削減できた残業時間: 月40時間 → 月10時間（▲75%）
  ・残業代削減額試算: ▲約¥○○万円/月
未達成・今後の課題:
  ・現場からのデータ入力（タブレット操作）に慣れるまで時間がかかった
  ・一部の設備（旧型NC機）はシステムと連携できず、手入力が残っている

出力形式:
・経営会議・補助金完了報告向けのサマリーレポート
・導入前後の比較表
・「当初目標に対する達成率」の評価（目標数値と実績の対比）
・「今後の展開（次のDXステップ）」の提案
```

## 社内DX委員会の議事録を作るプロンプト

DX推進委員会・IT活用検討会議の議事録を整理します。

```
以下のメモをもとに、DX推進委員会の議事録を作成してください。

【会議の基本情報】
開催日: （例：2026年6月25日 13:00〜14:30）
場所: （例：会議室A）
出席者: （例：社長・製造部長・品質保証部長・総務部長・IT担当 ○○（計5名））
議題: （例：生産管理システム導入の是非の検討）

【議論の内容（メモ）】
（例：
・社長から「今年度中にIT導入補助金を活用してシステムを入れたい」との方針
・製造部長: 「現場のベテランがIT機器を使えるか心配。操作研修が必要」
・品質保証部長: 「トレーサビリティの強化という意味では歓迎。
  ただしシステム導入後の運用ルールを早めに整備してほしい」
・総務部長: 「補助金申請の手間が心配。専門家に依頼するか?」
・IT担当: 「A社・B社・C社から提案を受けており、来月デモを予定」
決定事項: 「来月のデモを経て8月中に発注先を決定する。
  IT担当がIT導入補助金の申請準備を開始する。」
）

出力形式:
・正式な委員会議事録フォーマット
・「審議事項 / 議論のポイント / 決定事項 / 担当者・期限」を分けて記載
・次回の議題（案）を末尾に
```
