製造業の新製品開発では、試作→量産移行までの各フェーズで設計審査（デザインレビュー: DR）を実施し、設計の妥当性・量産性・品質リスクを評価します。AIを使って審査準備書類・議事録・指摘事項の整理を効率化できます。

---

## DR（デザインレビュー）事前準備チェックリストを作るプロンプト

設計審査の前に確認すべき内容を整理します。

```
以下の情報をもとに、製造業の新製品開発における
設計審査（DR: デザインレビュー）の事前準備チェックリストと
審査に必要な書類一覧を作成してください。

【開発プロジェクトの概要】
製品: （例: 自動車用ブラケット部品（鉄鋼プレス成形）
  寸法: W150×H80×D30mm / 板厚2.3mm / 材質: SPFC440）
開発ステージ: （例: 試作2回完了 → 量産移行審査（DR3）を控えている）
顧客: （例: 自動車Tier1メーカー / 量産開始: 6か月後）
特殊特性: 締結孔の寸法（CC: 重要特性）・板厚（SC: 安全特性）

【DRで評価する主な項目（例）】
設計面:
  ・図面公差と加工能力の整合性（Cp≧1.33達成見込み）
  ・材料規格書・部品表（BOM）の確定状況
  ・顧客仕様書への適合確認（寸法・材料・表面処理）

製造準備面:
  ・製造工程フロー（最終版）の確認
  ・量産用金型・設備の手配状況
  ・作業指示書・QC工程表の初版作成

品質計画面:
  ・PPAPの提出レベルと顧客承認スケジュール
  ・初物品の検証計画（MSA・初期流動管理）

出力形式:
・DR事前準備チェックリスト（項目・担当部門・確認状況・証跡）
・「DR当日に机上に並べる書類一覧」
・「DR指摘事項管理表のフォーマット（指摘内容・対応策・期限・担当）」
・「量産移行審査（DR3）でよく問われる質問と回答例」5問
```

## 設計審査の議事録と指摘事項対応記録を作るプロンプト

DRの議事録と、指摘事項への対応を追跡する書類を整理します。

```
以下の情報をもとに、設計審査（DR）議事録と
指摘事項対応記録を整理してください。

【DR実施概要】
審査名: （例: 新型ブラケット 量産移行審査（DR3））
日時: （例: 2026年6月28日 14:00〜16:30）
参加者: （例: 開発部・製造部・品質部・調達部 / 総勢10名
  議長: 開発部長）
審査品: 試作2品・量産移行時のサンプル2品

【審査結果】
指摘事項（抜粋）:
  1. 試作品の穴径が公差内だが、量産時のバラつきを考慮した
     プロセス能力（Cp値）の確認データが不足
     → 担当: 品質部 / 期限: 2週間以内
  
  2. QC工程表のサンプリング頻度の根拠が不明確
     → 担当: 品質部・製造部 / 期限: 1週間以内
  
  3. 外注先（表面処理工程）への品質要求の伝達が書面で確認できない
     → 担当: 調達部 / 期限: 3週間以内

審査の結論:
  （例: 指摘事項3件の解消を条件として量産移行を承認。
    確認期限を設けて追跡する。）

出力形式:
・DR議事録のフォーマット（目的・参加者・審議内容・結論・残課題）
・「指摘事項追跡管理表」（指摘No.・内容・担当・期限・対応状況・
  完了確認者 の表形式）
・「是正完了後の再確認フロー（誰が・いつ・どのように確認するか）」
・「DRの記録保管ルール（保管期間・格納場所・更新管理）」の提案
```

## 設計変更管理書類（ECN/DCN）の整理プロンプト

DR後の設計変更を管理するための書類を整理します（設計変更通知書との連携）。

```
以下の状況での設計変更管理書類を整理してください。

【設計変更の背景】
変更理由: （例: 量産移行審査後に顧客からの設計変更依頼
  （図面改訂: 穴径 φ12.0±0.05 → φ12.0±0.03 に公差変更）
  変更理由: 顧客での組付け不良（ガタつき）が発生）

変更の影響範囲:
  ・図面: Rev. A → Rev. B への改訂
  ・QC工程表: サンプリング頻度の見直しが必要
  ・金型: 追加加工が必要（見積もり中）
  ・納期: 変更対応完了まで最短3週間

顧客への対応:
  ・変更承認をPPAPで再提出する必要があるか顧客に確認中
  ・中間報告（変更対応スケジュール）は3営業日以内に提出を求められている

出力形式:
・設計変更通知書（ECN）のフォーマット（変更理由・変更内容・
  影響範囲・スケジュール・承認者）
・「変更影響範囲チェックリスト（何を確認・更新する必要があるか）」
・「顧客への変更対応中間報告メール」の文面
・「変更対応完了後の再バリデーション計画」の骨子
・「顧客承認なしで変更を実施してしまった場合のリスクと対応」のコメント
```
