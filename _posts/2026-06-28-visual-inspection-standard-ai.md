製造業の品質管理で「外観検査は目視でやっているが、担当者によって合否判断がバラバラ」という問題は多くの工場で発生しています。外観検査基準書や限度見本の管理文書をAIで文章化・整理することで、基準の明確化と教育の効率化が図れます。

---

## 外観検査基準書の骨子を作るプロンプト

製品ごとの外観検査基準を文書化します。

```
以下の情報をもとに、外観検査基準書を作成してください。

【製品情報】
品番・品名: （例：プレス加工品 品番A-1234、材質SPCC、表面処理なし）
適用工程: （例：仕上げ・梱包前の最終外観検査）

【合否判定の基準（検査員が目視で確認する内容）】
- 傷・キズ: （例：長さ10mm・深さ0.1mm超の傷は不合格。製品裏面は長さ20mmまで可）
- バリ: （例：高さ0.3mm超のバリは全面不合格。バリ取り後再検査）
- 凹み・打痕: （例：φ3mm超の凹みは不合格。φ1mm以下は合否判定不要）
- 錆び・汚れ: （例：赤錆は不合格。油汚れは脱脂後再検査（1回のみ可））
- 変色・焼け: （例：加工熱による変色（茶色・青色）は不合格（引張強度への影響懸念））

【限度見本の状況】
（例：合格限度見本 3点・不合格見本 2点を保管・更新日: 2026年1月）

出力形式:
・外観検査基準書フォーマット（適用品番 / 検査項目 / 合否基準 / 判定方法）
・検査項目を「合格」「不合格」「再検査（条件付き）」の3分類で
・写真貼付スペース（「ここに限度見本写真を貼付」というプレースホルダー）
・改訂履歴欄・承認者欄
```

## 限度見本管理台帳を作るプロンプト

工場内で使用する限度見本（合否見本）の管理台帳を整理します。

```
以下の情報をもとに、限度見本管理台帳の記載内容を作成してください。

【限度見本の情報】
登録番号: （例：LS-2026-001）
対象品番・品名: （例：品番A-1234 プレス加工品）
見本の種類: （合格見本 / 不合格見本 / 限度（境界）見本 から選択）
今回の例: 合格見本（最大許容傷あり）・不合格見本（バリ超過）各1点

【保管情報】
保管場所: （例：品質管理室 棚A-2）
管理者: （例：品質保証部 ○○）
制作日: （例：2026年7月1日）
有効期限: （例：製品廃版まで、ただし毎年1月に状態確認）

出力形式:
・限度見本管理台帳の1レコード分（台帳に追記できる形式）
・「登録番号 / 品番品名 / 種類 / 保管場所 / 管理者 / 制作日 / 有効期限 / 状態確認記録」の欄
・破損・紛失・廃棄時の手順注記
```

## 外観検査員向けの教育テキストを作るプロンプト

新入りや異動者向けに外観検査の基準・やり方を説明するテキストを作ります。

```
以下の情報をもとに、外観検査員向けの教育テキスト（OJT用）を
作成してください。

【工場・製品の概要】
（例：精密プレス部品メーカー。SPCC・SUS等の板金プレス加工品を製造。
 主要客先は自動車部品メーカーで外観要求が厳しい）

【教育対象者】
（例：外観検査工程に初めて配属されたパート・アルバイト）

【教えたいこと】
1. 外観検査の目的（なぜやるか）
2. 基本的な検査のやり方（目の動かし方・照明の使い方）
3. 合否の判断基準（傷・バリ・汚れ等）
4. 合格品・不合格品の分け方・置き場所
5. 不合格品を発見した時の報告手順

出力形式:
・A4 1枚程度の教育テキスト（初日のOJTで渡せるレベル）
・専門用語はわかりやすく説明を添える
・最後に「確認テスト」を3問程度つける
```

## 外観不良の傾向分析コメントを作るプロンプト

月次または週次の外観不良の集計結果から、改善コメントを作ります。

```
以下の外観不良集計データをもとに、傾向分析コメントを作成してください。

【集計期間】（例：2026年7月度）

【外観不良の発生内訳】
- バリ: 8件（先月比+3件）/ 主な発生工程: プレス後バリ取り工程
- キズ: 3件（先月比-1件）/ 主な発生タイミング: 梱包作業時
- 汚れ: 2件（先月比変化なし）/ 主な内容: 作業時の油付着
- 変色: 1件（先月比±0）/ 原因: 加工熱

合計: 14件（先月比+2件）

出力形式:
・最も多い不良種別と発生傾向の要約（3〜5行）
・「改善が必要な優先度の高い項目」を1〜2点
・来月の改善アクション案（簡潔に）
```
