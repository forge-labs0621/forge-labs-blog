トヨタ生産方式（TPS/リーン製造）は「ムダ・ムラ・ムリ」の排除によってリードタイムを短縮し、生産性を高める考え方です。AIを使ってムダの特定・改善計画・かんばん設計・バリューストリームマッピング（VSM）の文書化を効率化できます。

---

## ムダの7種類を特定・整理するプロンプト

現場のムダを体系的に洗い出し、改善の優先順位を整理します。

```
以下の情報をもとに、製造現場の「ムダ」を
7つのムダ（TPS）の観点から整理してください。

【現場の状況（例）】
対象工程: （例: 機械加工ライン（旋盤→フライス→検査→梱包））
現在のリードタイム: 受注から出荷まで○日（目標: ○日以内）

【現場で気になっている点（現場観察・ヒアリング結果）】
・仕掛品在庫が工程間に多く、常に山積みになっている
・作業者が次の材料を取りに行く時間が頻繁にある
・工程①が終わっても工程②が混んでいて待ちが発生する
・製品の寸法不良があり、手直しが発生している
・検査工程で再検査が何度も行われている
・作業者が工具を探す時間が頻繁にある

7つのムダの分類（TPSの定義）:
  ① 加工のムダ: 品質に貢献しない過剰な加工
  ② 在庫のムダ: 必要以上の原材料・仕掛品・完成品の保管
  ③ 造りすぎのムダ: 必要以上・必要でない時期の生産
  ④ 手待ちのムダ: 作業者が次工程を待っている時間
  ⑤ 動作のムダ: 価値を生まない作業者の動き
  ⑥ 運搬のムダ: 必要でない材料・製品の移動
  ⑦ 不良・手直しのムダ: 不良品の発生と修正作業

出力形式:
・「7つのムダ分析シート（観察された事象→ムダの分類→発生原因→
  改善アイデア）」の表形式
・「優先度ランキング（リードタイム短縮への影響度×改善の容易さ）」
・「クイックウィン（すぐできる改善）と中期改善（設備・レイアウト変更）
  の分類」
・「改善活動の進め方（どの工程から着手するか）の論拠」
```

## かんばん（引き取りかんばん・生産かんばん）設計プロンプト

在庫を最小化しながら生産を滑らかにするかんばんシステムを設計します。

```
以下の情報をもとに、かんばん方式の設計と
運用ルールを整理してください。

【かんばんを適用したい品目・工程】
品目: （例: 自動車用ブラケット / 月産5,000個 / 1日250個）
工程フロー: プレス加工（内製）→ 在庫 → バリ取り（内製）→ 在庫 →
  検査（内製）→ 完成品在庫 → 顧客出荷

【各工程の情報】
プレス工程:
  ・1日あたりの生産能力: 400個/日
  ・段取り替え時間: 45分/回
  ・1ロットの最小生産数: 200個（段取り替えコストを考慮）

バリ取り工程:
  ・能力: 300個/日
  ・リードタイム（前工程から受け取り〜完了）: 1日

検査工程:
  ・能力: 350個/日
  ・リードタイム: 0.5日

出力形式:
・「かんばん枚数の計算（引き取りかんばん・生産かんばんそれぞれ）」
  （計算式と根拠の説明）
・「かんばんカードのフォーマット（品番・数量・工程・補充先の記載）」
・「かんばん運用ルール（いつ動かす・どこに置く・誰が管理）」
・「かんばん導入の手順（試行→修正→定着）」
・「在庫削減効果の試算（現状の仕掛品日数→目標日数）」
```

## バリューストリームマッピング（VSM）の現状分析を作るプロンプト

製品の流れを可視化し、改善ポイントを見つけます。

```
以下の情報をもとに、バリューストリームマッピング（VSM）の
現状図の整理と改善ポイントの特定を行ってください。

【対象製品・プロセス】
製品: （例: 精密機械部品（外径旋削品）/ 月産2,000個）

【各工程の情報（現状）】
工程: 材料受入 → 旋盤加工 → 検査 → 梱包 → 出荷

工程別データ:
  材料受入: リードタイム3日 / 在庫: 5日分 / バッチサイズ: 500個
  旋盤加工: サイクルタイム 3分/個 / 稼働率 80% / 仕掛在庫: 3日分
  検査: サイクルタイム 1分/個 / 仕掛在庫: 1日分
  梱包: サイクルタイム 0.5分/個 / 仕掛在庫: 0.5日分

顧客の要求:
  ・月産2,000個 / 週4回納品（月曜・火曜・木曜・金曜）
  ・顧客からの発注: 週次（毎週月曜日に翌週の発注書が来る）

現状の問題点:
  ・受注から出荷まで平均12日かかっている（顧客要求は7日以内）
  ・材料の長いリードタイムが問題の一因

出力形式:
・「VSMの現状図の要素整理（工程・在庫・情報フロー・リードタイム）」
・「プロセスタイム比率の計算（付加価値時間 / トータルリードタイム）」
・「改善ポイントの特定（どこが最大のムダか）」
・「VSM将来図の骨子（目標状態: リードタイム○日への改善計画）」
・「改善の実施優先順位と各施策の期待効果」
```
