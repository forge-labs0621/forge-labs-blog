製造業では改善活動・設備導入・ISO認証取得・工場移転など様々なプロジェクトが同時進行します。AIを使ってWBS（作業分解構造）・課題管理・進捗報告書・ステークホルダーへの説明資料を効率的に整理できます。

---

## プロジェクトのWBS（作業分解構造）を作るプロンプト

プロジェクトのタスクを細分化し、スケジュールを管理します。

```
以下のプロジェクト情報をもとに、WBS（作業分解構造）と
プロジェクト計画を整理してください。

【プロジェクトの概要】
プロジェクト名: （例: ISO 9001 新規認証取得プロジェクト）
目的: ISOの品質マネジメントシステムを構築し、
    ○年○月に審査を受けて認証取得する
プロジェクト期間: ○月○日 〜 ○年○月○日（○ヶ月間）
プロジェクトリーダー: 品質管理部 ○○さん
関与する部門: 品質・製造・設計・総務・経営

【プロジェクトの主要フェーズ】
Phase 1: 現状分析・ギャップ分析（ISO要求事項 vs 現状）
Phase 2: 文書化（品質マニュアル・手順書・記録フォーマット作成）
Phase 3: 教育・訓練（全社への品質方針・手順の周知）
Phase 4: 内部監査の実施と改善
Phase 5: 認証審査・審査対応

【制約条件】
・通常業務と並行して進めるため、月○時間以内/人
・外部コンサルなし（社内リソースのみ）

出力形式:
・「WBSの展開（フェーズ→作業→タスクの3階層）」
・「ガントチャートの骨子（主要タスクの開始・終了日・担当）」
・「プロジェクトのクリティカルパス（遅れると全体に影響する作業）」
・「リソース計画（誰がいつどのタスクを担当するか）」
・「週次の進捗確認ミーティングのアジェンダフォーマット」
```

## 課題管理表（アクションリスト）を整理するプロンプト

プロジェクトの課題・決定事項・宿題を一元管理します。

```
以下の情報をもとに、プロジェクトの課題管理表
（アクションリスト）を整理してください。

【直近の会議で出た課題・宿題（例）】
課題1: 品質マニュアルの「適用範囲」の定義が曖昧
  → ○○さんが適用範囲の案を来週金曜までに提案する

課題2: 外注先（A社・B社）への品質基準通知の方法が未定
  → 購買の○○さんが外注先との協議を来月中に完了させる

課題3: 初回内部監査の実施日程が調整できていない
  → ○○さんが3月第2週で全部門の日程調整をする

決定事項: 記録の電子化はフェーズ2には含めず、フェーズ3で対応

保留: ISO文書管理のクラウドシステムの選定
  → 費用対効果を確認してから判断（CEO承認が必要）

出力形式:
・「課題管理表のフォーマット（課題番号・課題内容・担当者・
  期限・状態[未着手/進行中/完了/保留]・備考）」
・「優先度付き課題リスト（期限が近いもの・影響度が大きいもの）」
・「完了した課題の記録方法（エビデンス保管のルール）」
・「次回ミーティングでの確認事項（課題の状況報告の進め方）」
```

## 経営者への進捗報告書を整理するプロンプト

プロジェクトの進捗を経営者・上司に報告します。

```
以下の情報をもとに、プロジェクト進捗報告書の
骨子を整理してください。

【プロジェクトの現状】
プロジェクト名: ISO 9001認証取得プロジェクト
報告時点: 開始から2ヶ月経過（全体の40%経過）

進捗サマリー:
  ・Phase 1（ギャップ分析）: 完了（予定通り）
  ・Phase 2（文書化）: 進行中（完成度60%）
    品質マニュアル: 完成
    主要手順書（5件）: 3件完成・2件作成中
    記録フォーマット: 未着手

健全性の評価:
  ・スケジュール: やや遅れ（手順書が当初予定より遅い）
  ・コスト: 予定内（外部コスト発生なし）
  ・品質: 文書の精度は高い（内部確認済み）

主要リスク:
  ・4月の繁忙期と文書化が重なるとリソース不足になる懸念
  → 3月中に文書化を完了させるか、一部外注を検討

経営者への要請:
  ・4月の繁忙期前に文書化を完了させるための残業費予算（約¥50,000）

出力形式:
・「1ページ進捗報告書（全体サマリー・フェーズ別進捗・
  リスク・経営者への要請）のフォーマット」
・「信号機方式のステータス表示（青=順調/黄=要注意/赤=問題）の活用」
・「リスクと対策を簡潔に説明するための表現例」
・「経営者が承認すべき事項の明確な提示方法」
・「次の30日間のアクションプランの骨子」
```
