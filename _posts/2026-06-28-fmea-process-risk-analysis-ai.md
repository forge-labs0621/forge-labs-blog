FMEA（Failure Mode and Effects Analysis / 故障モード影響解析）は、製品・工程の故障やリスクを事前に洗い出して、問題発生を予防するための分析手法です。AIを使ってFMEAの項目整理・RPN計算・改善計画の記述を効率化できます。

---

## 工程FMEA（PFMEA）の記入支援プロンプト

製造工程のリスクを体系的に分析します。

```
以下の情報をもとに、工程FMEA（PFMEA）の
分析シートの骨子を整理してください。

【分析対象の工程】
製品: （例: 自動車用プレス成形品 / 品番: AP-2025）
分析対象工程: プレス工程（ブランク抜き → 成形 → 穴あけ）

【各工程のFMEA分析対象】
工程①「ブランク抜き」:
  機能（この工程が行うこと）: 鋼板から所定寸法のブランク材を打ち抜く

故障モード候補（想定される不良）:
  ・寸法不良（ブランクサイズが規格外）
  ・バリの発生（型の摩耗・クリアランス不適切）
  ・位置ズレ（材料セット不良・送りミス）

工程②「成形」:
  機能: ブランク材を所定の形状に曲げ加工する

故障モード候補:
  ・曲げ角度不良（スプリングバック・材料ロット差）
  ・割れ（材料の伸び不足・曲げRが小さすぎる）
  ・表面疵（型の磨耗・潤滑不足）

出力形式:
・「PFMEA分析シートの構成（工程機能・故障モード・
  影響・重大度・発生原因・発生頻度・現行の管理方法・
  検出難易度・RPN）」
・「RPNの計算（重大度×発生頻度×検出難易度）と
  各スコアの評価基準」
・「高RPN項目の改善計画（予防処置・検出能力強化）」
・「AIAG-VDA 2019年版FMEA（ AP+AP方式）への対応のポイント」
・「FMEAを現場担当者と一緒に作るときの進め方」
```

## 設計FMEA（DFMEA）の記入支援プロンプト

製品設計段階でのリスクを分析します。

```
以下の情報をもとに、設計FMEA（DFMEA）の
分析シートを整理してください。

【分析対象の製品・機能】
製品: （例: 樹脂製ハウジング部品 / 電装部品収納用）
分析対象の機能・要求:
  ① 防水性の確保（IPX4 要求）
  ② 衝撃吸収（落下試験1m）
  ③ 寸法精度の確保（取付部分の精度 ±0.1mm）
  ④ 外観品質（ヒケ・ウェルドライン・シルバーストリークの防止）

【重点分析したい機能】
「防水性の確保（IPX4）」に重点を置いて分析する

関連する設計上の故障モード候補:
  ・シール面の寸法精度不足（成形収縮・金型精度）
  ・パーティングラインの食い違い（金型合わせ面の段差）
  ・ゲート位置の影響（ウェルドラインが防水シール部に発生）
  ・ヒケによるシール面変形

出力形式:
・「DFMEAシートの構成（設計機能・故障モード・潜在的影響・
  重大度・潜在原因・発生頻度・予防管理・検出管理・RPN）」
・「各故障モードのRPN計算と優先度の判定」
・「設計改善の提案（材質・構造・金型設計・成形条件の観点）」
・「DFMEAとPFMEAの関係（設計の故障モードが工程でどう影響するか）」
・「FMEAの見直しタイミング（設計変更・クレーム発生時の更新ルール）」
```

## FMEA改善計画と進捗管理のプロンプト

FMEAで特定した高リスク項目の改善活動を管理します。

```
以下の情報をもとに、FMEAの高RPN項目への
改善計画と進捗管理フォーマットを整理してください。

【FMEAから抽出された高RPN項目】
項目①: 「曲げ工程の角度不良（スプリングバック）」
  現在のRPN: 重大度7 × 発生頻度6 × 検出難易度4 = 168
  改善目標RPN: 100以下

  検討中の改善策:
    予防策: 材料ロットごとのスプリングバック量確認・型調整
    検出策: 初物確認でのデジタル角度計測（自動判定）

項目②: 「バリ発生（型摩耗による）」
  現在のRPN: 重大度5 × 発生頻度5 × 検出難易度5 = 125
  改善目標RPN: 80以下

  検討中の改善策:
    予防策: 型の定期点検周期の短縮（5,000ショット→3,000ショット）
    検出策: 型摩耗のモニタリングシートの強化

出力形式:
・「FMEA改善計画書（項目・改善内容・担当者・完了期限・
  改善後の想定RPN）の一覧フォーマット」
・「改善前後のRPN比較と効果検証の方法」
・「改善の進捗確認ミーティングのアジェンダ（誰が・何を・
  いつまでに報告するか）」
・「改善完了後のFMEA書類の更新手順」
・「FMEAを「絵に描いた餅」にしないための運用のコツ」
```
