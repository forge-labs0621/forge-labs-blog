## 点検記録、人によって書き方が違う問題

設備の定期点検記録は、トラブル時や監査の根拠資料になる大事な書類です。
ただ現場では「異音なし」とだけ書く人もいれば、細かく状況を書く人もいて、
記録の質が担当者によってバラバラになりがちです。

この「書き方のばらつき」を、AIに整形させることで揃える方法を紹介します。

## 試してみた例

点検した人がメモを残すだけにします。

```
点検日: 2026-06-21
設備名・設備番号: マシニングセンタ MC-03
点検者: 鈴木
点検項目ごとの結果: 異音なし、油量OK、フィルター汚れあり、安全カバー固定OK
異常の有無と対応: フィルターの汚れについて清掃済み
次回点検予定: 2026-07-21
```

「異常なしと異常ありを明確に区別する」「異常があれば対応状況を必ず明記
する」という条件を付けてAIに渡すと、次のような記録票になります。

```
点検日: 2026-06-21
設備名・設備番号: マシニングセンタ MC-03
点検者: 鈴木

点検項目と結果
| 点検項目 | 結果 |
|---|---|
| 異音 | なし |
| 油量 | OK |
| フィルター | 汚れあり |
| 安全カバー固定 | OK |

異常の有無と対応状況
フィルターの汚れを確認したため、点検時に清掃を実施し対応済み。
その他の項目に異常はなし。

次回点検予定
2026-07-21
```

誰が書いても同じ構成・同じ粒度の記録になるのが、テンプレート化の効果
です。

## 重大な異常の場合は別ルートで

このやり方はあくまで日常点検の記録整形が目的です。稼働停止が必要な
レベルの重大な異常が見つかった場合は、AIの下書きで済ませず、必ず人の
判断で責任者へ別途報告してください。AIは「記録を揃える」ためのもので、
「安全判断をする」ためのものではない、という線引きが大切です。
