製造業でも顧客情報・設計図面・製造ノウハウなど機密情報の保護が重要です。取引先からISMS認証・Pマーク取得を求められるケースも増えています。AIを使って情報セキュリティ管理体制の整備を効率化できます。

---

## 情報セキュリティ方針とリスクアセスメントを整理するプロンプト

自社の情報資産とリスクを体系的に整理します。

```
以下の情報をもとに、製造業の情報セキュリティ
方針とリスクアセスメントを整理してください。

【当社の情報資産の概要】
主な機密情報:
  ・顧客から受領した設計図面・仕様書（紙+電子）
  ・自社の見積書・原価データ
  ・顧客リスト・取引条件（受注残・納期・単価）
  ・製造技術ノウハウ・作業標準書
  ・従業員の個人情報（氏名・住所・給与情報）

情報システムの現状:
  ・会計ソフト: MF会計（クラウド）
  ・CAD: AutoCAD（ローカルインストール）
  ・ファイル共有: NASサーバー（社内のみ）
  ・メール: Gmail（無料版）
  ・PCのセキュリティ: Windows Defenderのみ

現在の課題:
  ・NASにパスワードなしでアクセスできる
  ・設計図面を個人のUSBで持ち帰る慣行がある
  ・顧客からISMSに準じた情報管理を求められている

出力形式:
・「情報セキュリティ方針文書（目的・適用範囲・基本原則・
  組織の責任）の骨子」
・「情報資産台帳のフォーマット（情報の種類・保管場所・
  アクセス権限・重要度・リスク）」
・「主要リスクのアセスメント（不正アクセス・情報漏洩・
  ランサムウェア・内部不正のリスク評価）」
・「リスク対応策の優先順位（コスト低×効果高のクイックウィン）」
・「ISMSまたはPマーク取得の検討ポイント（費用・期間・メリット）」
```

## 情報セキュリティ規程・社内ルール文書を作るプロンプト

従業員向けのセキュリティルールを文書化します。

```
以下の情報をもとに、製造業向けの情報セキュリティ
規程・社内ガイドラインを整理してください。

【規程に含めたい主要ルール】
①パスワード管理:
  ・パスワードは8文字以上・英数字記号混在
  ・3ヶ月に1回の変更（顧客要求による）
  ・他人との共有禁止・メモ書きの禁止

②設計図面・機密文書の取り扱い:
  ・顧客から受領した図面はNASの指定フォルダにのみ保管
  ・USBへのコピーは上長承認・使用後は即消去
  ・印刷時は必要最小限・使用後は鍵付きキャビネットに保管

③テレワーク（在宅勤務）:
  ・VPN経由でのアクセスのみ許可
  ・プライベートPCでの業務使用禁止
  ・家族が見える場所での機密情報閲覧禁止

④インシデント対応:
  ・情報漏洩・PCウイルス感染を発見した場合は即時に情報管理責任者に連絡
  ・自己判断での対処禁止（状況悪化を防ぐため）

出力形式:
・「情報セキュリティ規程の全文構成（目的・対象・
  定義・各種ルール・違反時の措置）」
・「従業員向け情報セキュリティガイドライン（A4 1〜2枚のコンパクト版）」
・「年次の情報セキュリティ教育の内容（研修教材の骨子）」
・「情報セキュリティチェックシート（年1回の自己点検用）」
・「誓約書フォーマット（入社時・定期確認時に署名させる書類）」
```

## 情報セキュリティインシデント対応書類を作るプロンプト

セキュリティ事故発生時の対応手順を整理します。

```
以下の情報をもとに、情報セキュリティインシデント発生時の
対応手順書と記録書類を整理してください。

【発生したインシデントの例】
ケース①（ランサムウェア感染）:
  状況: 従業員がメールの添付ファイルを開いたところ、
    PCがランサムウェアに感染した。NASに保存されている
    設計図面も暗号化されている可能性がある。

ケース②（情報漏洩疑い）:
  状況: 退職した元従業員が、在職中に顧客の設計図面を
    個人のクラウドストレージに保存していたことが判明した。
    顧客への影響範囲は現時点で不明。

出力形式（ケース①・②それぞれについて）:
・「インシデント発生直後の初動対応手順（発見から30分・2時間・
  24時間以内のアクション）」
・「影響範囲の調査方法と記録フォーマット（何が・どこまで・
  どの程度影響したか）」
・「顧客・取引先への報告の要否判断基準と報告文の骨子」
・「行政機関への届出（個人情報保護委員会への報告義務の確認）」
・「再発防止策の策定と報告書フォーマット」
・「バックアップからの復旧手順（ランサムウェア対応）」
```
