「担当者が変わると業務が回らなくなる」「どの部署が何をするか、誰も全体像を把握していない」——受注から出荷までの業務フローを標準化することで、引き継ぎ・新人教育・品質安定に繋がります。AIを使って業務フローの可視化・標準化書類を整備できます。

---

## 受注から出荷までの業務フロー整備書類を整理するプロンプト

```
以下の情報をもとに、受注から出荷までの
業務フロー図と業務標準書を作成してください。

【会社の概要と業務の現状】
業種: 精密部品の受託製造業（多品種少量）
従業員: 65名
主な課題:
  ①業務の流れが担当者の「頭の中」にあって文書化されていない
  ②「誰に聞けばいいか」「次にどうするか」が新人に伝わらない
  ③部門をまたぐ連携（受注→生産計画→製造→検査→出荷）でボトルネックが発生

【業務の大まかな流れ（ヒアリング結果）】
受注フェーズ（営業・受注担当）:
  見積依頼受信 → 図面確認 → 見積作成 → 見積提出 → 受注確認 → 受注登録

生産準備フェーズ（生産管理・技術）:
  受注登録 → 製造可否確認 → 生産計画立案 → 工程設計・段取り計画 → 材料手配

製造フェーズ（製造部）:
  材料入庫 → 作業指示 → 加工実施 → 工程間確認 → 完成

品質確認フェーズ（品質部）:
  完成品検査 → 検査記録 → 合否判定 → 合格品を出荷指示

出荷フェーズ（物流・出荷担当）:
  出荷指示 → 梱包 → 送り状作成 → 出荷 → 売上計上

【標準化の優先事項】
①受注登録から生産計画確定までの連携ルール（現在: 1〜3日かかる）
②検査合格後の出荷指示のタイミングと伝達方法
③顧客への出荷連絡の基準（いつ・誰が・何を連絡するか）

出力形式:
・「受注〜出荷までの業務フロー図（スイムレーン形式・部門×工程）」
・「各フェーズの担当者と役割一覧（RACI表）」
・「受注登録〜生産計画確定の標準手順書（1〜3日を1日に短縮する改善込み）」
・「出荷前チェックリスト（梱包・送り状・伝票・顧客連絡の確認事項）」
・「業務フロー引き継ぎマニュアル（新人・代行者がフローを追える説明書）」
```
