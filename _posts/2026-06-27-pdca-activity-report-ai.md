製造現場では品質改善・安全改善・生産性向上のためにPDCAサイクルを回すことが多いですが、「実際に活動はしているが記録が残っていない」「報告書の書き方がわからない」という現場担当者も少なくありません。AIを使うと、活動の記録・進捗報告・振り返りの文書化を効率化できます。

## 製造現場でのPDCAとは

| フェーズ | 製造現場での例 |
|---|---|
| **Plan（計画）** | 不良率を1%から0.5%に下げる目標を立て、原因分析・対策を計画する |
| **Do（実施）** | 作業手順書を改訂し、教育訓練を実施する |
| **Check（確認）** | 1ヶ月後の不良率を測定し、目標と実績を比較する |
| **Act（改善）** | 効果があった対策を標準化、効果がなかった対策を見直す |

## PDCA計画書（Planフェーズ）を作るプロンプト

---

```
以下の情報をもとに、製造現場のPDCA改善計画書を作成してください。

【改善テーマ】
（例：機械加工工程の不良率削減 / 5S活動の定着化 / 設備稼働率の向上）

【現状の問題（数値があれば数値で）】
（例：
・機械加工工程の不良率が月平均2.8%（目標1.0%）
・原因は工具の交換タイミングが担当者によって異なること
・月に3〜4件の不良品が工程外まで流出している
）

【目標（できるだけ数値で）】
（例：3ヶ月以内に不良率を1.0%以下にする）

【対策案（思いついたこと）】
（例：
1. 工具交換基準を工数（加工個数）で定める
2. 工具交換チェックリストを作り、記録を残す
3. 工具交換後の初物確認を義務化する
）

【担当者・期間】
（例：担当: 製造課 ○○、実施期間: 2026年7月〜9月）

出力形式:
・「PDCA改善計画書」として
・現状分析（問題の事実・数値）
・目標（SMART：具体的・測定可能・達成可能・現実的・期限付き）
・対策の内容（「何を・誰が・いつまでに」の3点セット）
・効果の確認方法と確認時期
・担当者・承認者欄（プレースホルダー）
```

## PDCA月次進捗報告書（Doフェーズ）を作るプロンプト

計画に対して「今月何をやったか」を報告する書類です。

```
以下の情報をもとに、PDCA活動の月次進捗報告書を作成してください。

【改善テーマ】
（例：機械加工工程の不良率削減）

【今月（〇〇月）に実施したこと】
（例：
・工具交換基準を作成し、全作業者に朝礼で説明した（7月10日）
・工具交換チェックリストを現場に設置した（7月12日）
・初物確認の手順を作業手順書に追記した（7月15日）
）

【今月の実績（数値）】
（例：不良率 2.2%（前月2.8%から0.6ポイント改善）、不良流出 1件（前月3件）)

【計画と実績の差（遅れや問題）】
（例：
・工具交換チェックリストは設置したが、記入漏れが3件あった
・全員がルールを理解するまでに時間がかかった
）

【来月の計画】
（例：
・チェックリストの記入率を毎週確認する
・朝礼で工具管理の重要性を引き続き周知する
）

出力形式:
・「PDCA月次進捗報告書」として
・今月の活動実績（日付付き）
・実績数値と目標の比較表
・問題・気づき
・来月の計画
・報告者・確認者・報告日
```

## PDCA振り返り報告書（Check/Actフェーズ）を作るプロンプト

活動期間終了後の振り返りと標準化の記録です。

```
以下の情報をもとに、PDCA活動の振り返り報告書（最終報告）を作成してください。

【改善テーマ・活動期間】
（例：機械加工工程の不良率削減 / 2026年7月〜9月（3ヶ月間））

【目標と最終実績の比較】
（例：
・目標: 不良率1.0%以下
・最終実績: 不良率0.8%（目標達成）
・不良流出件数: 3ヶ月間で1件（活動前は月3〜4件）
）

【有効だった対策】
（例：
・工具交換基準（個数管理）の導入が最も効果があった
・初物確認の義務化は流出防止に効果的だった
）

【あまり効果がなかった対策・残課題】
（例：
・チェックリストは最初の1ヶ月は記入漏れが多く、効果が出るまで時間がかかった
・残課題: 夜勤の担当者への教育がまだ不十分
）

【標準化・水平展開の計画】
（例：
・今回策定した工具交換基準を、他の工程（フライス・ボール盤）にも展開する
・10月中に他工程の手順書改訂を完了する
）

出力形式:
・「PDCA活動振り返り報告書」として
・活動概要・期間
・目標と実績の比較（表形式）
・有効だった対策・効果があった理由
・残課題と次のアクション
・標準化・横展開の計画
・作成者・確認者・承認日
```

## 短い「PDCA日報」形式を作るプロンプト

毎日の活動を短くPDCAで記録する場合に使えます。

```
以下のメモをもとに、PDCA形式の日次活動記録を作成してください。

【今日の活動メモ（箇条書きで構いません）】
（例：
・朝礼で工具交換ルールについて確認した
・午後に旋盤3台の工具交換チェックリストの記入状況を確認した
・1台で記入漏れを発見、担当者に確認したところ手順を覚えていなかった
→ 夕礼で再度説明した
）

出力形式:
・P（本日の計画）・D（実施したこと）・C（確認した結果）・A（改善・調整）の4項目で
・各項目を2〜3文でコンパクトに
・翌日への申し送り事項を最後に一行
```

---

## AIを使う際の注意

- PDCA計画書・報告書はチームの活動の記録です。AIに下書きを作成させた後、**活動の事実・数値が正確かどうか**を担当者が確認してください
- 数値の改善効果は、AIが計算した値ではなく、**実際に測定した値**を記載してください
- PDCA活動の記録は、ISO 9001などの品質マネジメントシステムの「継続的改善の証拠」として保管することが推奨されます
