IATF 16949は自動車部品サプライヤーに求められる品質マネジメント規格です。ISO 9001の要求事項に加え、自動車業界固有の要求（コアツール・CSR・顧客固有要求事項）への対応が必要です。AIを使って書類整備と審査準備を効率化できます。

---

## CSR（顧客固有要求事項）の対応状況を整理するプロンプト

トヨタ・ホンダ・デンソー等の顧客から来るCSRを整理します。

```
以下の情報をもとに、CSR（顧客固有要求事項）対応状況一覧表を
作成してください。

【自社の取引先と主なCSRの例】
取引先A（例: トヨタ系Tier1）:
  ・CSR文書名: ○○品質管理規程（最新版: ○○年○月）
  ・主な要求項目:
    - PPAP（生産部品承認プロセス）の実施義務
    - 特殊特性の明確化と管理計画への反映
    - 製品トレーサビリティの確保（ロット単位）
    - 年1回の内部監査と報告書提出

取引先B（例: Tier2・海外自動車メーカー系）:
  ・CSR文書名: Quality Management Requirements Rev. X
  ・主な要求項目:
    - APQP（先行製品品質計画）の実施
    - SPC（統計的工程管理）データの提供
    - 8Dレポートの提出（不具合発生時、72時間以内）

【確認したいこと】
・IATF 16949の本文要求 vs 各顧客CSR要求の関係
・自社の現在の対応状況（対応済み / 対応中 / 未対応）
・審査員から確認される可能性が高い証跡

出力形式:
・CSR対応状況一覧表（顧客別・要求項目別）
・「IATF 16949本文との対応関係」（審査時の質問想定）
・「未対応項目のリスク評価と優先順位」
・「CSR改訂時の更新確認フロー（社内手順の骨子）」
```

## コアツール（APQP/PPAP/FMEA/SPC/MSA）の書類整備プロンプト

IATF 16949で求められるコアツールの書類を整理します。

```
以下の情報をもとに、IATF 16949のコアツール書類整備状況を
確認・整理してください。

【対象製品・プロセス】
（例: 自動車用プレス加工部品 / 年間生産量: 10万個
    主要工程: プレス→バリ取り→表面処理→検査→梱包）

【コアツールの整備状況】
1. APQP（先行製品品質計画）:
   現状: （例: 量産品については試作フェーズでの活動記録があるが、
     正式な「APQP計画表」「タイムライン」として文書化されていない）

2. PPAP（生産部品承認プロセス）:
   現状: （例: 顧客から求められた際のみ実施。レベル3の証跡を
     提出した経験はあるが社内での保管・管理が属人的）

3. FMEA（故障モード影響解析）:
   現状: （例: DFMEA（設計）は顧客が主導。PFMEA（工程）は
     QC工程表と連動させているが最終版の管理ができていない）

4. SPC（統計的工程管理）:
   現状: （例: 一部の特殊特性についてXbar-R管理図を
     作成しているが、管理限界の根拠が不明）

5. MSA（測定システム解析）:
   現状: （例: 実施したことがない / Gage R&Rのデータはある）

出力形式:
・各コアツールの「審査員が重点確認するポイント」
・「未整備書類のギャップ分析と優先度付け」
・「PPAP提出パッケージの必要書類一覧（Level 1〜3別）」
・「FMEA最新版の管理と変更履歴の保管方法の提案」
```

## IATF 16949 内部監査の準備書類を作るプロンプト

第三者審査（外部監査）に備えた内部監査計画と証跡を整理します。

```
以下の情報をもとに、IATF 16949の内部監査計画と
審査準備書類を整理してください。

【審査の種類と時期】
・次回の第三者審査（外部監査）: （例: 2026年○月予定 / 更新審査）
・内部監査の実施状況: （例: 年2回実施・直近は3か月前に完了）
・審査対象: プロセス別監査（製造プロセス/品質マネジメント/
  サポートプロセス）

【前回の審査での指摘事項】
（例:
  軽微不適合1件: 顧客クレームのトレンド分析が四半期単位で
    実施されておらず、経営レビューへのインプットが不十分
  観察事項2件: SPC管理図の管理限界見直しの根拠記録、
    外部提供者（外注先）への品質要求の伝達記録
）

【内部監査で重点的に確認したいプロセス】
（例: 製造プロセス・受入検査・外部提供者管理・製品リリースプロセス）

出力形式:
・内部監査チェックリスト（プロセス別・IATF 16949箇条番号付き）
・「前回指摘事項の是正状況確認フォーム」
・「審査員からよく聞かれる質問と模範回答例」（10問程度）
・「審査当日の証跡準備リスト（どのファイルをどこから出すか）」
・「審査の流れ説明資料（従業員向け・審査への心構え）」の骨子
```

## 特殊特性・重要特性の管理書類を作るプロンプト

安全・機能に重大な影響を与える特殊特性の管理を整理します。

```
以下の情報をもとに、特殊特性（SC/CC）の管理書類を
整理してください。

【自社製品の特殊特性の例】
SC（安全特性）:
  ・ブレーキ部品の板厚寸法（公差: ±0.05mm / Cp≧1.67要求）
  ・ねじ締結トルク値（管理値: ○〜○N・m）

CC（重要特性）:
  ・表面処理膜厚（規格値: ○〜○μm）
  ・溶接強度（引張強度: 〇〇MPa以上）

【現在の管理方法】
・QC工程表に記載あり
・SPC（Xbar-R管理図）で一部管理
・特殊特性のシンボルマーク（△・◎等）は図面・工程表に表示済み

出力形式:
・「特殊特性管理一覧表」のフォーマット（品番・特性名・管理値・
  モニタリング方法・記録場所・担当者）
・「特殊特性のサンプリング頻度と根拠の文書化方法」
・「工程異常発生時の特殊特性への影響確認フロー」
・IATF審査で「特殊特性の管理方法を説明してください」と
  聞かれた際の回答例
```
