ISO 9001取得・維持をしている製造業では、年1回の内部監査が必要です。監査前の準備書類（チェックリスト・スケジュール・被監査部門への通知）や、監査後の不適合対応書類の文書化をAIで効率化できます。

---

## 内部監査スケジュール・通知文書を作るプロンプト

内部監査の日程を決め、被監査部門に通知する文書を作ります。

```
以下の情報をもとに、ISO 9001内部監査の計画通知書を作成してください。

【基本情報】
実施年度: （例：2026年度）
監査の目的: （例：ISO 9001:2015に基づく年次内部監査）
監査実施期間: （例：2026年10月1日〜10月31日（1か月間））
主任監査員: （例：品質保証部 ○○（内部監査員資格保有））

【監査スケジュール（部門別）】
- 10月5日（月）: 製造部（午前9時〜12時）/ 監査員: ○○
- 10月12日（月）: 品質保証部（午前9時〜12時）/ 監査員: ○○
- 10月20日（火）: 購買部・営業部（午後13時〜15時）/ 監査員: ○○
- 10月27日（火）: 管理部・全体まとめ / 監査員: ○○

出力形式:
・各部門長への内部監査実施通知書
・「被監査部門への準備依頼事項」を含める（確認したい文書・記録の準備等）
・監査の流れ（オープニング→書類確認→現場確認→クロージング）の説明
・問い合わせ先（主任監査員の連絡先のプレースホルダー）
```

## 部門別内部監査チェックリストを作るプロンプト

部門ごとの監査で確認する事項をリスト化します。

```
以下の情報をもとに、○○部門向けの内部監査チェックリストを
作成してください。

【対象部門】
（例: 製造部（プレス・溶接・仕上げ工程））

【ISO 9001の関連条項（確認したい内容）】
・8.5.1 生産・サービス提供の管理
  （作業標準書の整備状況・現場での遵守状況）
・8.6 製品・サービスのリリース
  （検査基準書・検査記録の整備）
・8.7 不適合アウトプットの管理
  （不適合品の識別・隔離・記録）
・7.1.5 監視・測定のための資源
  （測定器の校正記録・使用状況）
・7.2 力量
  （作業員の教育訓練記録・適切な人員配置）

出力形式:
・「確認項目 / 確認方法（書類確認・現場確認・インタビュー） / 
  ISO条項 / 確認結果（良好・改善必要・不適合） / コメント」の表形式
・各項目15〜20件程度（重要項目を漏らさない量）
・確認するサンプル書類の例を各項目に添付
```

## 内部監査の不適合報告書を作るプロンプト

監査中に見つかった不適合事項を記録する書類を作ります。

```
以下の情報をもとに、内部監査の不適合報告書を作成してください。

【不適合の内容】
監査日・対象部門: （例：2026年10月5日 / 製造部）
監査員: （例：品質保証部 ○○）

不適合事項:
【不適合1】
  ・ISO条項: 7.1.5（監視・測定のための資源）
  ・不適合の内容: ノギス3本のうち2本の校正期限（2026年9月）が切れた状態で
    現場で使用されていた。校正記録台帳の確認により発見。
  ・不適合の証拠: 校正記録台帳（番号NG-02, NG-03）

【不適合2】（複数の場合）
  ・ISO条項: 7.2（力量）
  ・不適合の内容: 外注先から受け入れた新規作業者1名のOJT記録が
    未作成だった（教育訓練計画書の対象者リストに記載あり）
  ・不適合の証拠: OJT記録台帳（○○の記録なし）

出力形式:
・不適合報告書のフォーマット（不適合番号 / 監査日・部門 / 条項 / 
  不適合内容 / 証拠 / 被監査部門リーダーサイン欄）
・各不適合に「是正措置期限（例: 1か月以内）」の欄
・「軽微な不適合（OFI: 改善の機会）」と「不適合（NC）」を区別
```

## 是正措置計画・完了報告を作るプロンプト

内部監査の不適合に対する是正措置の文書を作ります。

```
以下の情報をもとに、内部監査不適合への是正措置計画書を作成してください。

【不適合の内容】
不適合番号: （例：NC-2026-001）
不適合内容: （例：ノギス2本の校正期限切れ状態での使用）

【原因分析（なぜ起きたか）】
直接原因: （例：校正管理台帳の確認を担当者が月次で行うことになっていたが、
  今月は確認を忘れていた）
根本原因: （例：校正期限のアラート機能が台帳になく、人的確認に依存していたため
  定期的な確認が漏れやすい仕組みだった）

【是正措置の内容】
即時対応（実施済み）: 
  （例：校正期限切れの2本を直ちに使用禁止。翌日に校正業者へ送付）
恒久対策:
  （例：校正管理台帳に「期限3か月前通知」の列を追加。毎月1日に
  QC会議で台帳確認することをアジェンダに組み込む。担当者を変更）
再発防止:
  （例：測定器台帳の管理を個人から部門管理に変更）

出力形式:
・是正措置計画書（不適合番号 / 原因 / 是正措置 / 担当 / 期限）
・「効果確認」欄（措置実施後○か月後にフォローアップ審査で確認する）
・完了確認者サイン欄
```

## 内部監査のまとめ報告書（経営層向け）を作るプロンプト

内部監査の結果を経営層・マネジメントレビューに報告する文書を作ります。

```
以下のデータをもとに、内部監査結果の経営報告を作成してください。

【今年度の内部監査実績】
実施期間: （例：2026年10月）
監査対象部門数: （例：4部門）
総確認項目数: （例：60項目）

【主な結果】
不適合件数: （例：2件（NC）/ 改善の機会: 3件（OFI））
主な不適合:
  - 測定器の校正期限管理の不備（製造部 / 是正措置対応中）
  - OJT記録の未作成（製造部 / 是正措置完了）
前年比: （例：前年は不適合4件→今年は2件に改善）

【良かった点（ポジティブな所見）】
（例：
・作業標準書の整備状況が改善された（全工程で作成完了）
・5S活動の定着が確認できた（前年OFIが解消）
）

【来年度の重点監査テーマ】
（例：外注管理（仕入先評価の定期実施）を重点テーマとする予定）

出力形式:
・マネジメントレビュー向けのサマリー（A4 1枚以内）
・「是正措置の進捗状況」を明示
・「品質マネジメントシステムの有効性への評価（高/中/低）」と根拠
```
