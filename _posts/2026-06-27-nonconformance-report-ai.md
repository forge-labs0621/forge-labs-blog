製造現場で品質異常が発生したとき「何が起きたか」「なぜ起きたか」「どう対処したか」を記録する「品質異常報告書（不適合品報告書・品質不具合報告書）」は、再発防止とISO対応の両面で重要な文書です。AIを使うことで、異常発生直後でも素早く整理した報告書を作れます。

## クレーム対応報告書との違い

| | 品質異常報告書 | クレーム対応報告書 |
|---|---|---|
| 対象 | **社内で発見**した不良・異常 | **顧客から連絡**があった不具合 |
| 目的 | 工程内の問題把握・再発防止 | 顧客への説明・信頼回復 |
| 報告先 | 社内（製造部長・品質部門） | 顧客 + 社内 |
| 文体 | 社内向け・技術的 | 顧客向け・丁寧 |

品質異常報告書は「社内で素早く原因を分析して手を打つ」ための文書です。

## 品質異常報告書を作るプロンプト

---

```
以下の情報をもとに、製造現場の品質異常報告書を作成してください。

【異常の種類】
（例：寸法不良、外観不良（キズ・打痕）、異物混入、機能不良　など）

【発生工程・発見場所】
（例：機械加工工程 → 工程内検査で発見 /
       最終検査で全数確認時に発見 /
       出荷後に顧客から連絡）

【発生日時・発見日時】
（例：2026年7月3日 午前10時頃 加工中に発見）

【不良品の数量・影響範囲】
（例：当該ロット100個中5個に不良を確認。
       他ロットへの影響：確認中）

【異常の内容（具体的に）】
（例：外径φ25.0±0.05mmの部品が、5個とも φ24.85〜24.88mmと
       規格下限を下回っていた）

【発生時の状況（わかる範囲で）】
（例：前日に工具を交換した。交換後の試し切りは確認したが、
       その後の加工分は未確認だった）

出力形式:
・「品質異常報告書」として
・発生日・発見日・報告者
・異常の種類・発生工程
・不良内容の詳細（事実のみ。推測は「推定」と明記）
・影響範囲・数量
・暫定処置（即時対応内容）
・原因調査中の場合は「調査中」として記載
・承認欄（プレースホルダー）
```

## 原因分析（なぜなぜ5回）をAIで整理するプロンプト

品質異常の「なぜ」を深掘りする際に使えます。

```
以下の品質異常について、「なぜなぜ分析」を行ってください。

【品質異常の事実】
（例：機械加工工程で、外径寸法が規格外れの部品が5個発生した）

【現時点でわかっていること】
（例：
・工具を前日に交換した
・交換後の試し加工は行ったが、合格確認後は寸法確認を行っていなかった
・工具の種類は合っていた
・材料の品番は正しかった
）

なぜなぜ分析を5段階で行い、
「真の原因（根本原因）」と「是正措置の方向性」を示してください。

出力形式:
なぜ1→なぜ2→なぜ3→なぜ4→なぜ5（真の原因）
各段階に「事実の確認が必要な点」があれば記載してください。
```

## 是正措置・再発防止策を文書化するプロンプト

原因が判明した後、「再発防止策」と「水平展開」を記録します。

```
以下の品質異常について、是正措置報告書を作成してください。

【品質異常の内容】
（例：工具交換後の寸法確認漏れによる外径不良）

【根本原因（なぜなぜ分析の結果）】
（例：工具交換後の加工再開手順に「寸法確認」の手順が明記されておらず、
作業者の判断に任されていた）

【実施した是正措置】
（例：
1. 工具交換手順書に「交換後の試し加工→寸法確認→合格確認後に本加工再開」を明記した
2. 工具交換チェックリストを作成し、確認欄に署名を必須とした
3. 全作業者に朝礼で手順書の改訂を周知した
）

【水平展開先】
（例：機械加工工程の他の設備（旋盤・マシニング）でも同様の手順書を改訂）

【再発防止効果の確認方法・時期】
（例：是正措置実施から1ヶ月後に、工具交換時の確認記録を抜き取り確認する）

出力形式:
・「是正措置報告書」として
・是正措置の内容（事実・日付付き）
・水平展開の内容と対象
・有効性確認の方法・期日
・担当者・確認者欄（プレースホルダー）
```

## 不適合品の処置（廃棄・手直し・特採）を記録するプロンプト

発生した不良品をどう処置したかも記録が必要です。

```
以下の不適合品の処置について、処置記録を作成してください。

【不適合品の内容】
（例：外径規格外れの加工品 5個）

【処置の内容（選択してください）】
□ 廃棄（全数スクラップ）
□ 手直し加工（修正後再検査）
□ 特別採用（特採）：顧客・上長の承認を得て使用

【処置の詳細】
（例：5個全数廃棄。廃棄処理は当日中に完了。
代品は同日中に優先加工で手配）

【処置完了日】
（例：2026年7月3日）

出力形式:
・処置区分・数量・処置内容・処置日・担当者欄
・特採の場合は「承認者・承認日・適用条件」を追加
```

---

## 品質異常報告書を書く際の注意

- **事実と推測を明確に分ける**: 「〜だったため（確認済み）」と「〜と推定される（要確認）」を区別して記載
- **数値で書く**: 「大幅に」「少し」ではなく具体的な数値（例：5個中5個・外径 -0.15mm）
- **顧客名・機密品番はAIに入力しない**: 社内報告書であっても注意が必要
