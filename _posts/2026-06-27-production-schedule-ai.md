製造業の生産管理担当者が日常的に悩むのが「スケジュールの変更・割り込み・遅延が起きたときの関係者への共有」です。AIを使うと、工程の整理・進捗報告・納期変更の連絡文を素早く作ることができます。

## AIが役立つ生産管理の場面

| 場面 | AIでできること |
|---|---|
| 工程整理 | バラバラなメモ・口頭情報を工程順に整理 |
| 進捗報告書 | 遅れの状況・原因・挽回策を文書化 |
| 納期変更連絡 | 顧客・社内への連絡文の下書き |
| 週次生産会議の準備 | 各工程の状況を箇条書きでまとめる |
| 工程ボトルネック分析 | 情報を整理して問題点を見えやすくする |

> AIはスケジュール計算（工数・リードタイム・着工日の計算）を正確に行うのは苦手です。**計算は生産管理ソフトや担当者が行い、文書化・整理をAIに任せる**という役割分担が最適です。

## 受注から出荷までの工程を整理するプロンプト

担当者から集めたバラバラな情報を工程表のベースにする場合に使えます。

```
以下のメモをもとに、受注から出荷までの工程を順番に整理してください。

【受注情報】
- 製品名・品番: （例：プレス加工部品 ABC-001）
- 受注数量: （例：200個）
- 客先希望納期: （例：2026年8月10日）

【各工程の担当者からのメモ（順不同で構いません）】
（例：
・材料調達: 既存在庫で対応可。発注不要（総務・田中）
・プレス工程: 段取り1時間＋加工2時間/50個。月水金が空いている（製造・鈴木）
・バリ取り・仕上げ: プレス後すぐできる。約30分/50個（製造・佐藤）
・検査: 全数外観検査。1時間/200個（品質・山田）
・梱包・出荷: 出荷前日の午後に実施（物流・川口）
）

出力形式:
・「No. / 工程名 / 担当者 / 作業内容・所要時間 / 着手可能条件」の表で整理
・工程の順番を論理的に並べる
・工程間の「待ち時間」や「確認事項」があれば明記する
・現時点でわかっていない情報（着工日・完了予定日など）は「要確認」と記載する
```

## 生産進捗報告書（遅延あり）を作るプロンプト

工程が遅延した場合、状況・原因・挽回策を上司や顧客に報告する文書が必要です。

```
以下の情報をもとに、生産進捗報告書（遅延報告）を作成してください。

【受注情報・当初計画】
- 製品名・品番: （例：金属フレーム XY-100）
- 受注数量: （例：500個）
- 当初出荷予定日: （例：2026年7月25日）

【遅延の状況】
- 遅延している工程: （例：溶接工程）
- 遅延の原因: （例：溶接装置が7月10日に故障。修理完了見込みは7月14日）
- 現在の完了見込み: （例：7月18日に溶接完了。出荷予定日は8月1日になる見込み）
- 挽回策: （例：後工程（研削・検査）を残業対応で圧縮予定。
         外注先（○○工業）に一部外注することも検討中）

出力形式:
・「生産進捗報告書（遅延報告）」として
・遅延が発生した工程・原因の説明
・現状の進捗（完了工程・残工程）
・修正後の出荷予定日
・挽回策
・今後の対応スケジュール
・担当者・確認者・報告日欄
```

## 納期変更の社内・顧客連絡文を作るプロンプト

遅延が確定した場合の関係者への連絡文を下書きします。

**社内連絡（上司・営業への報告メール）:**

```
以下の状況について、上司・社内営業担当へのメール（社内報告）の
下書きを作成してください。

【状況】
- 製品名・品番: （例：特注プレス部品 C-200）
- 客先: （例：株式会社○○製作所）
- 当初納期: （例：2026年7月20日）
- 変更後の予定納期: （例：2026年8月5日（約2週間の遅延））
- 遅延原因: （例：材料メーカーの供給遅延による材料入庫が2週間遅れた）
- 挽回策: （例：入庫次第、優先で加工・出荷する）

トーン: 事実を正確に・簡潔に。謝罪は一言添える程度で良い（社内メールのため）。
```

**顧客への連絡メール:**

```
以下の状況について、顧客（発注者）へのお詫び・納期変更連絡メールの
下書きを作成してください。

【状況】
- 顧客社名・担当者名: （例：株式会社○○製作所 資材部 山田様）
- 製品名・品番・数量: （例：特注プレス部品 C-200 / 300個）
- 当初納期: （例：2026年7月20日）
- 変更後の納期: （例：2026年8月5日（約2週間の遅延））
- 遅延原因: （例：材料メーカーの供給遅延のため）

トーン: 丁寧にお詫びを伝えつつ、原因と対応を明確に説明する。
過剰な謝罪表現は避けて、誠実で簡潔な文章にする。
```

## 週次生産会議の議題・資料を整理するプロンプト

毎週の生産会議の準備資料としても使えます。

```
以下のメモをもとに、今週の生産会議（週次）の資料を作成してください。

【各工程の状況メモ（担当者から集めたもの）】
（例：
・加工: 受注A-001（300個）は予定通り進行中。今週中に完了見込み。
         受注B-002（100個）は材料不足で着工待ち。
・検査: 先週からの在庫品（200個）の検査完了。合格率99%。
・出荷: 今週出荷予定は3件。月曜・水曜・金曜。
・受注: 新規引合いが2件。来週中に見積を出す予定。
・設備: 旋盤1台が木曜に定期メンテ予定。半日稼働停止。
）

出力形式:
・「週次生産会議 資料」として
・「先週の振り返り / 今週の重点課題 / 工程別状況 / 今週の出荷予定 / 設備・人員の注意事項 / 来週の見通し」の構成で
・箇条書きで簡潔に
```

## 工程ボトルネックの整理に使えるプロンプト

どこが詰まっているかを整理するだけでも、改善の議論のたたき台になります。

```
以下の生産状況から、ボトルネック（一番遅れている工程・滞留しがちな
箇所）を特定し、問題点を整理してください。

【各工程の処理能力・現在の状況】
（例：
・受注: 月30件受注中。最近は急増傾向。
・設計・見積: 設計1名対応。ピーク時は2週間待ち。
・材料調達: 通常1週間。欠品なし。
・加工: 3台稼働。設備の段取り替えに時間がかかる（平均2時間/件）。
・検査: 1名担当。検査に3日かかることも。受注増加で追いつかない。
・梱包・出荷: 問題なし。
）

出力形式:
・ボトルネックと思われる工程を優先順位をつけて挙げる
・各ボトルネックの「なぜ滞留するか」の原因仮説を1〜2個
・短期改善案（すぐできること）と中期改善案（設備・人員の検討が必要なもの）を分けて提示
・あくまで「整理と仮説」であり、現場の実態確認が必要なことを明記する
```

## AIを使う際の注意

- 工程表の日程（着工日・完了日）の計算はAIに任せると誤りが生じることがあります。日程計算は生産管理ソフトまたは担当者が行い、文書化だけをAIに任せる分担が推奨です
- 顧客への連絡文は必ず担当者・上司が内容を確認してから送信してください
- 生産スケジュールは機密性の高い情報を含む場合があります。外部AIサービスへの入力時は、顧客名・品番・数量等の具体的情報をマスクすることを検討してください
