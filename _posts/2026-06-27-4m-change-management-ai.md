製造現場で品質に影響する変更が生じたとき、それを記録・管理する「4M変更管理」は、ISO 9001やIATF 16949などの品質マネジメント規格で要求される重要な仕組みです。「変更のたびに書類を作るのが手間」「何を書けばいいか迷う」という担当者向けに、AIを使って4M変更管理記録を素早く作る方法を紹介します。

## 4M変更管理とは

4Mとは製造の4要素:

| M | 内容 | 変更の例 |
|---|------|---------|
| Man（人） | 担当者・技能者の変更 | 作業者の交代、新人配置、外注先の変更 |
| Machine（機械） | 設備・工具の変更 | 機械の更新、治具の変更、新型設備の導入 |
| Material（材料） | 原材料・部品の変更 | 仕入先変更、材質変更、代替品への切り替え |
| Method（方法） | 作業手順・工程の変更 | 加工条件の変更、手順書の改訂、工程の順番変更 |

これらのうち品質に影響しうる変更が起きた際に「変更届」や「変更影響評価」を記録し、顧客や上長の承認を得てから変更を実施する、というのが4M変更管理の基本的な流れです。

## 4M変更届をAIで作るプロンプト

---

```
以下の変更内容をもとに、4M変更管理届を作成してください。

【変更の種類（4Mのどれか）】
（例：Machine（機械）、Material（材料）　など）

【変更前の状態】
（例：旧型プレス機A型・プレス能力100t）

【変更後の状態】
（例：新型プレス機B型・プレス能力120t・省エネタイプ）

【変更の理由】
（例：旧機器の老朽化による更新、仕入先廃業による代替品への切り替え　など）

【変更予定日】
（例：2026年8月1日）

【品質への影響の有無・影響内容】
（例：プレス能力が上がるため加工条件（プレス圧・速度）を再設定する必要がある）

【検証・確認事項】
（例：変更後に試作品を10個作り、寸法・外観・強度を確認する）

出力形式:
・タイトル「4M変更管理届」
・変更区分（Man/Machine/Material/Method）
・変更前・変更後の状態（並べて比較できる形式）
・変更理由
・品質への影響評価（影響あり/影響なし・影響内容）
・変更前の確認事項・試作・検証計画
・承認欄（申請者・確認者・承認者・日付）
```

## 出力例（材料変更の場合）

---

**4M変更管理届**

変更区分：Material（材料）  
申請日：2026年7月○日  
申請者：○○ / 確認者：□□ / 承認者：△△

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| 材料名 | 鋼材 SS400（仕入先A社） | 鋼材 SS400相当品（仕入先B社） |
| 変更理由 | 仕入先A社の廃業により代替品への切り替えが必要 | |
| 変更予定日 | 2026年8月1日 | |

**品質への影響評価**  
影響の有無：あり  
影響内容：材料の製造ロットが変わるため、機械的特性（引張強度・伸び）を確認する必要がある

**変更前の確認事項・検証計画**  
1. 仕入先B社の材料規格書・試験成績書を取得し、A社品と規格値を比較する  
2. 初回入荷ロットから試験片を採取し、引張試験を実施する  
3. 試作品○個を製造し、外観・加工性を確認する  
4. 検証完了後、本格切り替えを実施する

## 変更影響評価のプロンプト

変更が品質・コスト・納期にどう影響するかを評価する際にも使えます。

```
以下の変更について、品質・コスト・納期・安全への影響を評価してください。

【変更の概要】
（例：外注先Aを廃業により外注先Bに切り替える。加工工程は同じだが
設備・作業者が異なる）

【現在の品質基準】
（例：寸法公差 ±0.1mm、表面粗さ Ra1.6以下、出荷検査合格率 99%以上）

評価してほしい観点:
1. 品質リスク（変更によって品質が悪化する可能性があるか）
2. コストへの影響（単価・段取り費用等）
3. 納期への影響（切り替え時の生産中断はあるか）
4. 安全への影響（作業者・設備に関わるリスク）

出力形式:
・各観点ごとに「影響あり/影響なし・影響内容・対策」の3列
・総合評価（実施可/条件付き実施可/実施不可）
・実施条件がある場合、条件の内容
```

## 変更後の効果確認記録を作るプロンプト

変更を実施してから一定期間後に「変更前後で品質・効率が変わったか」を確認する記録も重要です。

```
以下の情報をもとに、4M変更後の効果確認記録を作成してください。

【変更内容の概要】
（例：プレス機を更新、材料を代替品に切り替え）

【変更前の実績（数値）】
（例：不良率 2.0%、稼働率 85%）

【変更後の実績（数値）】
（例：不良率 1.5%、稼働率 92%）

【確認期間】
（例：変更後2週間・変更後1ヶ月）

【想定外の問題・気づき】
（例：切り替え当初は材料の扱いに慣れず不良が出たが2週目以降は安定）

出力内容:
・変更前後の比較表（数値）
・効果のコメント（良かった点・残課題）
・今後の対応方針（追加で観察が必要な項目・改善活動）
・確認者・承認者・確認日
```

## 4M変更管理をAIで効率化するポイント

4M変更管理でAIが特に役立つのは以下の部分です。

**AIに任せてよい作業**
- 変更前・変更後の対比表の整形
- 影響評価の観点（品質/コスト/納期/安全）の漏れチェック
- 文章の整理・敬語への変換

**人が確認すべき作業**
- 変更が本当に品質に影響するかどうかの技術的判断
- 試作・検証の合否判定
- 顧客・上長への報告・承認プロセス

ISO 9001・IATF 16949では「変更の管理」として記録の保存が求められます。AIで下書きを作成し、担当者が確認・修正した上で保管する形が実用的です。
