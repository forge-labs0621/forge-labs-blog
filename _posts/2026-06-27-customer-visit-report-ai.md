取引先への訪問後に社内へ提出する「訪問報告書」や「商談メモ」は、情報を整理して文章化するのに時間がかかる書類です。AIを使うと、訪問後のメモや会話の記録をもとに、正式な報告書を素早く作れます。

## AIが役立つ訪問報告書・商談メモの場面

| 訪問の種類 | 主な報告内容 |
|---|---|
| 営業訪問・商談 | 受注状況・新規引合・クレームの有無・競合情報 |
| 品質問題の対応訪問 | クレーム内容・先方の要求・対策の説明 |
| 購買・調達訪問（仕入先） | 供給状況・価格交渉・新製品の確認 |
| 初回訪問（新規開拓） | 相手企業の概要・ニーズ・次のステップ |
| 外注先の現地確認 | 工場状況・品質管理体制・改善指示 |

## 営業訪問報告書を作るプロンプト

```
以下の訪問メモをもとに、社内向け訪問報告書を作成してください。

【基本情報】
訪問日: （例：2026年7月8日（火））
訪問先: （例：株式会社○○製作所 資材部）
訪問者（自社）: （例：営業部 田中○○）
面談相手: （例：資材部長 山田様、資材課長 佐藤様）
訪問目的: （例：7月受注状況の確認・品質問題フォローアップ）

【面談の内容（メモ）】
（例：
・7月受注内示: 300個（先月比 +50個）。追加受注の理由は競合の納期遅延。
・品質問題のフォローアップ: 先月の寸法外れについて、
  対策説明を行い了解いただいた。今後は月次で不良率の報告を求められた。
・新製品の話: 来期（10月〜）に新型エンジン部品の外注先を探している。
  単価・加工仕様の詳細は8月上旬に先方より資料送付予定。
・競合情報: 先月まで一部を△△工業に発注していたが、品質問題で見直し中とのこと。
・次回訪問: 8月7日に新製品の詳細打ち合わせ
）

出力形式:
・「取引先訪問報告書」として
・訪問日・訪問先・面談者・訪問目的
・面談内容の要点（受注状況・品質・新案件・競合情報等を区別）
・アクションアイテム（誰が・何を・いつまでに）の表
・次回訪問予定
・提出先・提出日・作成者
```

## 品質問題対応訪問の報告書を作るプロンプト

クレーム対応・品質問題の説明のために取引先を訪問した際の報告書です。

```
以下の状況をもとに、品質問題対応訪問の報告書を作成してください。

【基本情報】
訪問日: （例：2026年7月5日（土））
訪問先: （例：株式会社□□自動車部品 品質保証部）
自社出席者: （例：品質部長 鈴木、製造課長 中村）
先方出席者: （例：品質保証部長 木村様、品質保証課 高橋様）

【クレームの内容（事前把握していた内容）】
（例：6月出荷の溶接品に寸法外れが発見。先方検査で10個/500個が不合格。
クレームは書面で受領済み。）

【訪問での内容（メモ）】
（例：
・自社より対策報告書を提出し、原因（溶接治具の摩耗）と対策（治具交換、
  検査頻度UP）を説明した。
・先方からの指摘: 対策は了解。しかし「今後同じことが起きたときに
  どう検知するか」の仕組みを7月中に提示してほしいとの要望。
・返品・賠償: 不合格品10個は自社で回収し再加工または交換で対応で合意。
  追加賠償は不要との確認が取れた。
・今後のモニタリング: 8月出荷分から1ヶ月間、検査証明書の添付を求められた。
）

出力形式:
・「品質問題 対応訪問報告書」として
・クレーム内容・訪問の経緯
・面談での説明内容と先方の反応
・合意事項（返品・賠償・今後の対応）
・自社へのアクションアイテム（担当者・期限付き）
・上長への報告事項（緊急度・影響範囲）
・作成者・報告先・作成日
```

## 外注先・仕入先の現地確認報告書を作るプロンプト

外注先の工場を訪問して品質管理体制や生産状況を確認した際の報告書です。

```
以下の情報をもとに、外注先 現地確認報告書を作成してください。

【基本情報】
訪問日: （例：2026年7月3日（木））
訪問先: （例：△△精工株式会社（外注先・旋盤加工））
自社確認者: （例：品質部 田中）
先方案内者: （例：製造部長 佐藤様）

【確認した内容（メモ）】
（例：
・作業環境: 工場は清潔に保たれており、5Sは良好。
・設備の状態: 旋盤4台のうち3台は近年更新済み。1台は古く、次年度更新予定とのこと。
・品質管理: 検査は目視と測定器。全数検査は実施しておらず抜き取り検査（5%）。
  ただし弊社向け品については、依頼に応じて全数検査が可能とのこと。
・在庫・納期対応力: 材料在庫は1週間分。急な受注増には追加発注が必要。
・スタッフ: 作業員15名（うち経験5年以上は10名）。技術力は問題ない印象。
・課題: 測定器の校正記録が一部不整備。改善依頼をした。
）

出力形式:
・「外注先 現地確認報告書」として
・確認日・確認先・確認項目と評価（良好/要改善/要フォロー）
・改善依頼事項（何を・いつまでに改善してもらうか）
・総合評価コメント（取引継続の観点から）
・次回確認予定・フォロー方法
```

## 新規取引先への初回訪問報告書を作るプロンプト

展示会・紹介等で知った会社への初回訪問後の社内報告書です。

```
以下のメモをもとに、新規取引先への初回訪問報告書を作成してください。

【基本情報】
訪問日: （例：2026年7月10日（金））
訪問先: （例：○○電子株式会社 購買部）
自社訪問者: （例：営業部 山田）
面談相手: （例：購買部長 高橋様、購買課 斉藤様）
きっかけ: （例：展示会で名刺交換後、先方から連絡いただいた）

【面談内容（メモ）】
（例：
・先方の概要: 電子機器（産業用制御機器）の製造業。従業員200名。
  金属部品の調達先を増やしたいとのこと。
・ニーズ: プレス加工品（SUS304、薄板0.5mm〜2mm）を月300〜500個規模。
  精度: ±0.05mm以上が要求されることも多い。
・現状の課題: 現在の調達先は1社のみで、品質問題と納期遅延が続いている。
・自社のアピール: 精密プレス加工の実績・短納期対応を紹介。先方から興味を持っていただいた。
・次のステップ: 試作品の見積もり依頼をいただける予定。8月上旬に図面送付とのこと。
）

出力形式:
・「新規取引先 初回訪問報告書」として
・訪問先の概要（業種・規模・ニーズ）
・面談内容の要点
・自社への関心度・受注可能性の評価（高/中/低）
・次のアクション（見積もり準備・フォロー時期等）
・商談の優先度（緊急性・規模感から）
```

## AIを使う際の注意

- 訪問報告書は「事実の記録」が基本です。AIが整理した内容に、実際の発言や決定事項と異なる点がないか確認してください
- 取引先から聞いた「競合情報」「取引条件」「発注予定」などは機密情報として社内限りで扱い、外部AIサービスへの入力時は注意してください
- 訪問報告書は早期に提出するほど情報の鮮度が高いです。訪問当日または翌日中の提出を推奨します
