製造業でAI（ChatGPT）を使い始めると「どのプロンプトをどの書類に使えばいいか」で迷うことがあります。この記事では書類の種類別にプロンプトの選び方と使い分けを解説します。

---

## 書類の「目的」で選ぶ

製造業の書類は大きく「記録する」「報告する」「管理する」「改善する」の4種類に分けられます。

### 記録系書類のプロンプト

**向いている書類:** 作業日報・点検記録・生産実績・ヒヤリハット

**基本のプロンプト構造:**
```
【記録した内容】
（今日行った作業・確認した内容をそのまま書く）

上記の内容を、以下のフォーマットで整理してください：
・日時: 
・作業者: 
・作業内容: 
・確認結果（問題あり/なし）: 
・特記事項: 
・次回対応: 
```

**ポイント:** 記録系は「事実」をそのまま書き、AIに「整理・フォーマット化」だけさせるのが原則。AIに事実を作らせてはいけません。

---

### 報告系書類のプロンプト

**向いている書類:** 月次報告書・管理会議資料・顧客クレーム報告書・監査報告書

**基本のプロンプト構造:**
```
以下のデータをもとに、○○報告書を作成してください。

【データ・事実】
（数値・事象を箇条書きで）

【報告先】
（例: 経営者 / 顧客の品質部門 / 社内の部門長）

【報告の目的】
（例: 月次の進捗確認 / クレームの原因と対策の説明）

出力形式: A4 1〜2枚相当の報告書形式
```

**ポイント:** 報告先と目的を明示することで、文章の「難易度・詳細度・語気」が自動的に調整されます。

---

### 管理系書類のプロンプト

**向いている書類:** 台帳・チェックリスト・手順書・管理計画書

**基本のプロンプト構造:**
```
以下の条件で【書類名】の管理フォーマットを作成してください。

管理対象: （何を・何個・誰が管理するか）
管理項目: （必ず含めたい列・項目）
更新頻度: （日次/週次/月次/年次）
確認者: （誰がチェックするか）
保管期間: （法的要求または社内ルール）

使いやすいExcelまたは表形式で出力してください。
```

**ポイント:** 「管理しやすさ」と「情報の網羅性」のバランスが重要。項目が多すぎると現場で使われなくなります。

---

### 改善系書類のプロンプト

**向いている書類:** カイゼン提案書・不適合報告書・是正処置報告書・品質改善計画

**基本のプロンプト構造:**
```
以下の問題について、改善書類を作成してください。

【問題の事実】
発生日: 
発生場所・工程: 
内容: 

【現状の影響】
（コスト・品質・安全・顧客への影響）

【根本原因（わかっている範囲で）】

出力形式: 
・直接原因（表面的な原因）
・根本原因（なぜその直接原因が起きたか）
・是正処置（恒久的な解決策）
・有効性の確認方法
・再発防止策と担当・期限
```

## 「相手」で選ぶ

書類の受け取り手によって、プロンプトに指定すべき事項が変わります。

| 相手 | プロンプトに追加する指示 |
|------|------------------------|
| 社内（現場作業者） | 「専門用語を避け、小学生でもわかる言葉で」 |
| 社内（経営者） | 「エグゼクティブサマリーを先頭に、数値とポイントのみ簡潔に」 |
| 顧客（国内） | 「丁寧な敬語・ビジネス文書形式で」 |
| 顧客（海外・英文） | 「Please write in clear business English」 |
| 行政機関 | 「法令の様式・用語に準拠して正式な文体で」 |
| ISO審査員 | 「ISO要求事項の用語を正確に使用し、要求事項番号を記載して」 |

---

## 「緊急度」で選ぶ

書類の緊急度によって、プロンプトの使い方が変わります。

**急いでいる場合（今すぐ出す書類）:**
```
【状況】
（1〜2行で事実を簡潔に書く）

この状況を顧客に説明する緊急連絡メールを
3行以内で書いてください。詳細は後送とします。
```

**じっくり作る場合（重要な書類・品質が大事）:**
```
（詳細な情報をすべて提供）

上記をもとに、以下の手順で書類を作成してください：
Step1: 構成案を提示（承認後に本文作成）
Step2: 本文の初稿作成
Step3: 初稿を批判的に見直し、改善点を3つ指摘
Step4: 改善した最終版を出力
```
