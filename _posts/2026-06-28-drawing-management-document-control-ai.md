製造業における図面・技術文書の適切な管理は品質管理の根幹です。最新版の把握・版管理・配付先管理・設計変更の記録をAIで効率化することで、旧版図面による製造ミスを防止できます。

---

## 図面番号体系を整理するプロンプト

自社に合った図番ルールを設計します。

```
以下の情報をもとに、製造業の図面番号（図番）
体系の設計と管理ルールを整理してください。

【当社の現状と課題】
現状: 図面番号のルールが設計者ごとにバラバラ
  ・設計者Aは「PA-001-001」形式
  ・設計者Bは「20260601-01」（日付+連番）形式
  ・図面番号が重複しているケースが発生
  ・古い図面と新しい図面の区別が難しい

会社情報:
  ・業種: 精密板金加工（受託製造・自社製品両方あり）
  ・製品の種類: 受注品（客先図面あり）・自社標準品・特注品
  ・使用CAD: SolidWorks（3Dモデル）+ AutoCAD（2D）

管理したい情報:
  ・製品分類（受注品 / 自社標準品 / 試作品）
  ・製品番号（親番）・部品番号（子番）・材料種別
  ・改定版（A, B, C... 又は Rev.1, Rev.2...）
  ・作成日・最終更新日・作成者

出力形式:
・「図面番号体系の設計（フォーマット案 2〜3案の比較）」
・「各フォーマット案のメリット・デメリット」
・「既存の図面番号との移行方法（段階的移行の進め方）」
・「図面台帳のフォーマット（図番・品名・版数・
  作成日・改訂日・作成者・管理場所の管理表）」
・「CAD/PDMシステムでの管理との連携ポイント」
```

## 図面の版管理と設計変更記録プロンプト

設計変更時の版管理を確実に行います。

```
以下の情報をもとに、図面の版管理と
設計変更記録の管理方法を整理してください。

【設計変更の経緯（実例）】
図面番号: PA-0203-001-A（プレスブラケット 品番P-2203）
変更前: Rev.A（2025年3月作成）
変更内容:
  ・穴径: φ8.0 → φ8.5mm に変更（顧客指示）
  ・板厚: 1.6mm → 2.0mm に変更（強度不足のため）
  ・表面処理: 三価クロメート → アルマイト処理に変更
変更理由: 顧客からの設計変更指示（ECN番号: ECN-2026-045）
変更後: Rev.B（2026年6月1日）
影響範囲:
  ・現在の在庫品（Rev.A で製造済み品 50個）の扱い
  ・購買中の材料（2.0mm板は未発注のため問題なし）
  ・工具・治具の変更（穴明けドリルのサイズ変更が必要）

出力形式:
・「設計変更通知書（ECN）の記載内容の整理
  （変更内容・変更理由・影響範囲・対応期限・関係部署）」
・「版管理のルール（Rev.AからRev.Bへの切り替え手順）」
・「既存在庫品（Rev.A品）の処理方法の判断ガイド
  （使用可能/要改造/廃棄の判断基準）」
・「設計変更が関係する全部門への周知方法
  （設計→製造→品質→購買→顧客の連絡フロー）」
・「設計変更履歴の記録台帳フォーマット
  （図番・変更日・変更内容・版数・影響部署・完了確認）」
```

## 技術文書の配付・閲覧管理を整理するプロンプト

図面の配付先と閲覧権限を管理します。

```
以下の情報をもとに、技術文書（図面・
仕様書・手順書）の配付・閲覧管理体制を整理してください。

【管理対象の技術文書】
  ①客先図面（顧客から支給された図面）: 機密レベル: 極秘
  ②自社設計図面: 機密レベル: 社外秘
  ③工程手順書・QC工程表: 機密レベル: 社内限
  ④カタログ・会社案内: 機密レベル: 一般

【現状の問題点】
  ・客先図面がPCの共有フォルダに無制限に保存されていて
    誰でも閲覧・コピーできる
  ・図面の最新版と旧版が混在しており、旧版で製造する
    ミスが年2〜3件発生している
  ・退職した従業員が持ち出した図面の把握ができていない

【使用できるシステム】
  ・Windowsファイルサーバ（共有フォルダ）
  ・Google Workspace（Googleドライブ）
  ・紙での管理（印刷した場合）

出力形式:
・「技術文書の機密レベル別の管理ルール（閲覧権限・保管場所・
  コピー可否・持ち出しルール）」
・「図面管理台帳（配付先・配付日・版数・返却状況）の整備方法」
・「旧版図面の回収・廃棄手順と記録の残し方」
・「共有フォルダの権限設定の推奨構成
  （部門別フォルダ・アクセス権の設定例）」
・「技術文書管理規程の骨子
  （目的・対象文書・管理責任者・配付・改訂・廃棄の各規定）」
```
